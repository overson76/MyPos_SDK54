# MyPos 매장 운영 매뉴얼

> **이 문서는 매장 사장님 / 직원이 직접 보고 트러블슈팅·운영하기 위한 가이드입니다.**
> 기술적 상세는 `CLAUDE.md` 참조.

---

## 1. 빠른 시작 — 처음 사용

### 사장님 (대표) 가입

1. 폰 또는 PC 카운터에서 라이브 URL / 앱 첫 화면 → **"새 매장 만들기"**
2. 매장 상호 + 본인 표시 이름 입력
3. **매장 코드 자동 생성** (예: `RDJX-JA2Z`)
4. 이 코드를 직원에게 전달 (카톡 / 문자 / 입으로)

### 직원 가입

1. 첫 화면 → **"매장 참여"**
2. 사장님이 알려준 매장 코드 입력 + 본인 표시 이름
3. 사장님 폰 / PC 에 가입 요청 알림 도착 → 사장님이 **"승인"** 누르면 자동 입장

> **주의 — 익명 uid 는 (기기 + 브라우저) 단위**
> 새 폰 / 새 PC / 새 브라우저 / 데이터 클리어 시 → **재가입 필요**.
> 한 번 가입하면 그 기기에서는 영구 유지.

## 2. 일상 사용 흐름

### 주문 받기

| 흐름 | 동작 |
|---|---|
| 손님 입장 → 메뉴 정함 | 테이블 탭 → 빈 테이블 선택 |
| 메뉴 선택 | 카테고리 → 메뉴 → 수량 → "주문" |
| 추가 주문 | 같은 테이블 다시 선택 → 메뉴 추가 → "주문" (변경분만 자동 재고지) |
| 조리 완료 | 주방에서 조리완료 표시 → 테이블 강조 |
| 결제 | 테이블 → "결제하기" 또는 주문 화면 → "선불"/"후불" → **결제수단 선택** |

### 결제수단 4종 + 미분류

- **현금** / **카드** / **계좌이체** / **지역화폐** — 매일 일계 정산용
- **미분류** — 결제수단 모를 때 (옛 데이터 / 현장 결정 안 됨). 회계 사무소 송부 시 분류

> **결제 후 영수증 자동 출력** (PC 카운터 .exe 환경 한정)
> 결제 모달 안의 토글 → ON 시 결제 직후 프린터로 자동 발행. 사장님 선호 영구 저장.

### 매장 옮김 / 새 PC / 직원 폰 교체

→ **재가입 필요**. 새 기기에서 매장 코드 입력 → 사장님 승인 → 정상 동작.

## 3. 자주 묻는 문제 (FAQ)

### Q1. 가입 요청이 사장님 폰에 안 보여요

**1차 확인**: 사장님 폰의 **관리자 → 시스템 → 진단 / 운영 정보** 카드.
- "✓ 정상 — 이 기기가 대표(owner) 로 등록되어 있습니다." → 데이터 정상. 직원 폰 인터넷 / 앱 재시작 점검.
- "⚠ stores.ownerId 와 실제 owner 멤버 uid 가 어긋남" → 옛 가입 데이터 잔재. 새 매장 만들기로 복구 권장.
- "⚠ owner 권한 멤버가 0명" → 데이터 손상. Firebase 콘솔 직접 확인 필요.

**2차 확인**: 이 기기 ID (uid 끝 12자리) 와 대표 ID 끝 자리가 같은지.

### Q2. 매장 매출 합계가 안 맞아요

**확인 순서**:
1. 관리자 → 수익현황 → 오늘 결제수단별 카드 row → 사장님 시재와 비교
2. 부가세 분리 표시 토글 ON → 공급가액 + 부가세 = 합계 검증
3. 최근 30건 이력 → 누락된 결제 / 미분류 항목 확인

### Q3. 회계 사무소에 어떻게 보내나요?

**PC 카운터 (라이브 URL 또는 .exe) 에서**:
1. 관리자 → 수익현황
2. 상단 **"CSV 익스포트"** → "이번 달" 클릭
3. 다운로드 폴더에 `mypos-month-YYYYMMDD.csv` 저장
4. 이메일 / 카톡으로 회계 사무소 송부

CSV 컬럼: 시점 / 테이블 / 메뉴 / 결제수단 / 결제상태 / 배달주소 / 합계 / 공급가액 / 부가세

### Q4. 영수증 프린터 어떻게 연결?

**현재 상태**: 코드 준비됨, 실 프린터 결정 시 환경변수 한 줄로 활성화.

**도입 단계**:
1. 프린터 모델 결정 (Bixolon / Epson / Star — 80mm 서멀 권장)
2. 네트워크 (IP) 또는 USB 결정 — IP 추천 (도입 쉬움)
3. PC 카운터의 .exe 를 환경변수와 함께 실행:
   ```
   MYPOS_PRINTER_MODE=network
   MYPOS_PRINTER_HOST=192.168.1.100
   MYPOS_PRINTER_PORT=9100
   ```
4. 결제 모달의 "결제 후 영수증 자동 출력" 토글 ON
5. 첫 영수증 시범 출력 — 한글 / 정렬 / 절단 정상인지 확인

### Q5. 5대 동시 동기화가 안 돼요

**원인 후보**:
1. 인터넷 연결 — 모든 기기가 같은 매장의 Wi-Fi 또는 LTE
2. Firebase 한도 초과 — 무료 한도(100만 read/일) 가까이 가면 동기화 지연
3. **익명 uid 어긋남** — 위 Q1 진단 카드 확인

**복구 (최후 수단)**: 새 매장 만들기 → 5대 모두 새 코드로 재가입. 기존 매출 history 는 그대로 남음 (각 기기별).

## 4. PC 카운터 .exe 운영

### 처음 설치

1. 사장님 / 직원이 **GitHub Releases** 에서 .exe 다운로드:
   - `MyPos-1.0.X-x64.exe` (portable — USB 이동 가능)
   - `MyPos Setup 1.0.X.exe` (NSIS installer — 시작메뉴 / 바탕화면 바로가기 자동)
2. SmartScreen 경고 → **"추가 정보 → 실행"** 한 번 클릭으로 우회
3. 키오스크 모드 자동 적용 (풀스크린 + 메뉴바 X)

### 자동 업데이트

- 부팅 시 자동 확인 → 새 버전 있으면 백그라운드 다운로드
- **영업 중에는 강제 재시작 X** — 사장님이 자연스럽게 .exe 닫는 시점 (영업 종료) 에 다음 시작 시 새 버전
- 수동 확인: 관리자 → 시스템 → "🔄 자동 업데이트" 카드 → "지금 확인"

### .exe 종료 단축키 (키오스크 모드)

키오스크에서는 메뉴바 / X 버튼 X. 종료:
- **Alt + F4** — 표준 Windows 종료
- 작업표시줄 우클릭 → 종료
- 또는 관리자 권한으로 작업관리자 → MyPos.exe 종료

> Phase 2.2 에서 운영 UI 에 "앱 종료" 버튼 추가 예정.

## 6. 매장 환경 권장 사양

### PC 카운터

- Windows 10 / 11
- 터치 모니터 권장 (없어도 OK)
- 인터넷: 유선 우선 (Wi-Fi 보조)
- 사양: 일반 사무용 PC면 충분 (.exe 200MB + Chromium 베이스)

### 폰

- iOS 14 이상 / Android 7 이상
- 매장 Wi-Fi 또는 LTE 안정

### 영수증 프린터 (도입 시)

- 80mm 서멀, ESC/POS 표준
- IP 모델 추천 — 매장 라우터에 IP 고정 후 환경변수 설정
- 모델 예: Bixolon SRP-330II / Epson TM-T20III / Star TSP100

## 7. 백업 / 데이터 안전

| 영역 | 방식 |
|---|---|
| 매장 데이터 (주문 / 메뉴 / 매출 / 멤버) | Firebase Firestore — 자동 백업 |
| 매출 CSV | 사장님이 매월 회계 사무소 송부 = 자체 백업 |
| 메뉴 이미지 | 폰/PC 의 IndexedDB / AsyncStorage. 데이터 클리어 시 사라짐 → 사장님 폰 1대는 데이터 클리어 X 권장 |
| PIN | 기기별 (자동 잠금) + 매장 공유 (수익 PIN) — Firebase / SecureStore |

## 8. 트러블슈팅 — 마지막 수단

**모든 것을 시도해도 안 되면**:

1. 5대 모두 앱 강제 종료 + 재시작
2. 한 대씩 인터넷 끊고 다시 연결
3. **새 매장 만들기** — 매장 코드 새로 발급 + 5대 모두 재가입
   - 옛 매출 history 는 각 기기에 남음
   - 새 매장은 빈 상태로 시작 → 옛 데이터 영향 X

**Firebase 콘솔 직접 확인**: 사장님이 직접 확인 어렵다면, 진단 화면의 ID 정보 + 함께 운영 도와주는 사람에게 전달.
