# CID(발신자 번호 표시) 상담 — 한 번에 통하는 스크립트

> 매장: LG U+ 센트릭스(Centrex, 기업용 인터넷전화) 가입 매장
> 목표: 매장 카운터 PC(MyPos.exe) 에서 SIP 서버로부터 CID 신호 수신
> 직전 진단: SIP REGISTER 시도 → **404 Not Found 응답** → 광/LTE 망분리 의심 확정

---

## 0. 왜 지금까지 상담이 안 통했나 — 핵심 오해

상담사들이 못 알아들은 이유는 **사장님이 "CID 가 안 돼요" 만 말씀하셔서**입니다. CID 는 한국 통신사 시스템에서 보통:

- **A. 070 인터넷전화 단말기 액정에 표시되는 발신번호**
- **B. ATA / IP 전화기에서 PC 로 전달되는 SIP INVITE 의 From 헤더**
- **C. 별도 CID 박스(시리얼/USB) → PC 로 전달되는 RS-232 신호**

매장에서 원하시는 건 **B 형태** — 즉 *LG U+ 센트릭스의 SIP INVITE 를 매장 PC 의 sip 클라이언트가 직접 받는 것*. 이걸 상담사에게 "**SIP 클라이언트 추가 등록**" 또는 "**SIP 콘텐츠 분배(Forking) 설정**" 이라고 정확히 표현해야 즉시 알아듣습니다.

## 5. 상담사 응답별 분기 — 어떻게 진행할지

### 응답 A: "PC 용 SIP 사용자를 추가로 등록해 드리겠습니다"
✅ **최선의 결과**. 추가 ID/PW 받음 → 매장 도착 시 사장님 PC 환경변수에 박으면 즉시 동작.
- 받아야 할 정보:
  - 추가 SIP 사용자 ID
  - SIP 비밀번호
  - SIP 도메인 (lgdacom.net 인지 확인)
  - SIP 서버 호스트 (현재 ATA `192.168.10.100` 사용할지, 다른 외부 SIP 프록시 사용할지)
  - 멀티 등록 시 INVITE 가 양쪽 단말에 동시에 가는지 / 순차로 가는지

### 응답 B: "센트릭스는 한 회선에 한 단말만 등록 가능합니다"
⚠️ SIP 멀티 등록 불가. 대안 요청:
- "그럼 **ATA 에서 발신자 번호를 별도 신호(시리얼/USB/HTTP/UDP 브로드캐스트)** 로 매장 PC 에 흘려보내는 옵션이 있나요?"
- "혹은 **CID 전용 박스**(이른바 'CID 어댑터') 별도 신청 가능한가요?"

### 응답 C: "ATA 의 SIP 프록시 / 멀티 디바이스 기능 확인해 보겠습니다"
⏳ 기술팀 콜백 요청 받음 — 시간대 / 회신 번호 명확히.

### 응답 D: "단말 펌웨어 / 설정 변경 출장 신청해 드리겠습니다"
⏳ 매장 출장 예약. 가능하면 **사장님 매장 영업 외 시간**으로.

### 응답 E: "ID/PW 가 임시 값이라 등록 거부됩니다"
❗ `tmpid7133` 같은 임시 ID 가 코드 default 인데 실제 매장 ID 는 다름. 정확한 ID/PW 받기 — 이게 가장 빠른 해결.

## 6. 통화 후 — 매장 현장 즉시 할 일

1. **받은 ID/PW/도메인을 매장 PC 환경변수에 박기** (관리자 권한 PowerShell):
   ```powershell
   setx MYPOS_SIP_HOST "<상담받은 IP 또는 도메인>"
   setx MYPOS_SIP_USER "<상담받은 ID>"
   setx MYPOS_SIP_PASS "<상담받은 PW>"
   setx MYPOS_SIP_DOMAIN "<상담받은 도메인 — 보통 lgdacom.net>"
   setx MYPOS_SIP_PORT  "5060"
   setx MYPOS_SIP_EXT   "<상담받은 내선/대표번호 끝자리>"
   ```
2. **MyPos.exe 완전 종료 후 재시작** (환경변수는 시작 시 1회만 로드).
3. **관리자 → 시스템 → CID 진단** (1.0.9 진단 빌드가 IPC 로 보여줌):
   - `registerSentCount` 가 1 이상으로 증가하는지
   - `lastResponseStatus` 가 **`200 OK`** 또는 **`401 → 200`** 인지 (404 / 503 / timeout 이면 실패)
4. **테스트 전화**: 본인 폰으로 매장 070 번호로 발신 → 화면 상단에 navy 배너가 뜨는지.
5. **다른 단말 충돌 없는지**: 기존 IP 전화기에서도 동시에 벨이 울리는지 / 한쪽만 받는지 확인.

## 7. 대안 — 상담사가 "절대 안 됩니다" 답하면

### 대안 1: 3rd party CID 박스 (RS-232/USB)
- 매장 전화선에 직접 물려서 발신번호를 PC 시리얼 포트로 흘려주는 하드웨어.
- 검색: "CID 모듈", "RS-232 발신번호 표시기" (가격 5~15만 원, 설치 간단).
- LG U+ 센트릭스 무관 — 일반 PSTN/070 회선 모두 호환.
- 우리 앱은 SIP 만 구현 — RS-232 받으려면 별도 모듈 추가 (1.0.x 추가 개발).

### 대안 2: 인포뱅크 / 무한CID 같은 클라우드 CID 서비스
- LG U+ 회선 그대로 두고, 별도 CID 서비스 사업자에 가입.
- HTTP webhook 또는 TCP 푸시로 매장 PC 에 발신번호 통보.
- 월정액 (1만 원 내외).
- 매장 PC 가 인터넷만 되면 동작 — 망분리 영향 없음.

### 대안 3: ATA → PC 같은 공유기 합류 (사장님 직전 결정)
- USB-LAN 어댑터 매장 카운터 PC 에 연결 → ATA 와 같은 공유기에 직접 LAN 케이블 연결.
- 이미 결정된 안 — LG U+ 도움 없이 사장님이 직접 가능.
- 그 후에 정확한 SIP ID/PW 받으면 즉시 동작.
