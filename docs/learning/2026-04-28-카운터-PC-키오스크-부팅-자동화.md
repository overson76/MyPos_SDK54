## 💻 PC 에서 마이포스를 띄우는 방식 — 3가지 옵션

데스크톱 개발 비유: **"AutoCAD 노트북에 큰 도면 모니터 + 디지타이저 펜 연결한 작업대"** 와 같은 구조. 본체(노트북) 안 보이고, 사용자는 모니터+터치로만 작업.

| 방식 | 동작 그림 | 작업량 | 1차 채택 |
|---|---|---|---|
| **A. 크롬 풀스크린** (`--kiosk`) | 노트북 부팅 → 크롬 자동 → URL → 풀스크린 | **거의 0일** (배포만) | ⭐ 1차 |
| **B. PWA** (Progressive Web App, 설치형 웹앱) | A + 작업표시줄 아이콘, 주소창 없음 | +0.5일 | ⭐ 운영 들어가서 추가 |
| C. Electron / Tauri 로 .exe | 진짜 데스크톱 앱 | +3~5일 | 1차에는 과함 |

→ **A → B 자연 승급**. C 는 영수증 프린터 / 금전함 같은 로컬 하드웨어 직접 제어가 필요해질 때 검토.

## 🔁 일상 사용 시나리오

```
[아침 영업 준비]
1. 노트북 전원 ON (뚜껑 닫혀있어도 외부 모니터/키보드/터치로 동작)
2. Windows 시작 → 시작프로그램의 크롬이 자동으로 mypos URL 열기
3. F11 풀스크린 (또는 PWA 면 자동 풀스크린)
4. 화면: 테이블 탭 — 1~N번 테이블 그리드

[손님 받음]
1. 1번 테이블 손님 → 화면에서 "1번" 터치
2. 주문 탭 자동 진입 → 메뉴 그리드에서 "김치찌개" 터치
3. 옵션(맵기/공깃밥) 선택 → "담기" 터치
4. 추가 메뉴 같은 식으로 담음
5. "주문 확정" 터치 → 1초 안에 직원 폰 + 주방 태블릿에 동일 주문 표시

[주방]
- 주방 태블릿: 주문현황 탭 켜놓음
- 새 주문 들어오면 화면 위쪽에 카드처럼 추가됨
- 조리 끝 → "준비 완료" 터치 → 카운터 화면에서도 상태 변경

[손님 결제]
1. 손님 카드 단말기로 결제 (지금처럼)
2. 결제 끝나면 카운터 화면의 1번 테이블 → "결제 완료" 터치
3. 테이블 비워짐, 매출 history 에 자동 기록

[영업 종료]
- 노트북 그냥 끔. 데이터는 클라우드(Firestore) 에 자동 저장
- 다음날 켜면 어제 매출/주문 기록 그대로 있음
```

## ⚙️ 부팅 자동화 — 키오스크 모드 셋업 (실무 권장)

수기 → POS 전환의 가장 큰 장벽이 **"매번 켤 때마다 뭐 눌러야 함"**. 이걸 없애는 3개 셋업:

### 1) 크롬 키오스크 모드 — 시작프로그램 등록

**무엇을**: 크롬이 풀스크린으로 마이포스 URL 만 띄우게.

```
chrome.exe --kiosk https://mypos-sdk54.overson76.workers.dev
```

| 플래그 | 효과 |
|---|---|
| `--kiosk` | 풀스크린 + 주소창 숨김 + 종료 차단 (Alt+F4 도 무시) |
| URL 직접 지정 | 크롬 시작 페이지 무시하고 바로 마이포스 |

**시작프로그램 등록**:
1. `Win+R` → `shell:startup` 입력 → 엔터 → 시작프로그램 폴더 열림
2. 빈 곳 우클릭 → 새로 만들기 → 바로 가기
3. 항목 위치: `"C:\Program Files\Google\Chrome\Application\chrome.exe" --kiosk https://<your-url>`
4. 이름: "MyPos 키오스크"
5. 저장

다음 부팅부터 자동으로 마이포스가 풀스크린으로 뜸.

**키오스크 종료 방법** (관리/디버깅 시):
- `Ctrl+Shift+Esc` → 작업 관리자 → 크롬 종료
- 또는 `Alt+F4` 안 먹으면 `Ctrl+F4` (탭 닫기) 시도

### 2) Windows 자동 로그인 (비밀번호 없이 부팅 → 바로 키오스크)

**무엇을**: 부팅하면 로그인 화면 없이 바로 데스크톱 → 시작프로그램 실행.

1. `Win+R` → `netplwiz` 입력 → 엔터
2. "사용자 이름 및 암호를 입력해야..." 체크박스 **해제**
3. 적용 → 비밀번호 두 번 입력 → 확인

**보안 주의**: 매장 노트북은 외부인이 만질 수 있음. 자동 로그인 = 누구나 켜면 사용 가능. 매장 운영 외 시간엔 노트북 잠그거나 끄기.

### 3) 절전 / 화면 끔 OFF

**무엇을**: 영업 중 손님 안 오는 시간에 화면 안 꺼지게.

`설정` → `시스템` → `전원 및 배터리`:

| 항목 | 값 |
|---|---|
| 화면 끄기 | **사용 안 함** (전원 연결 시) |
| 절전 모드 | **사용 안 함** (전원 연결 시) |
| 노트북 덮개 닫음 | **아무 작업도 안 함** (외부 모니터 사용을 위해) |

마지막 항목이 핵심 — 뚜껑 닫아도 외부 모니터로 계속 작동해야 함.

## ⚠️ 터치스크린 환경 주의 사항 (PC 만의 함정)

| 항목 | 이슈 | 대응 |
|---|---|---|
| **가상 키보드** | 메뉴 이름/메모 입력 시 폰처럼 자동으로 화면 키보드가 안 올라옴 | Windows "터치 키보드" 자동 호출 옵션 켜거나, 노트북 키보드도 나와있게 |
| **마우스 hover** | 터치엔 hover 가 없음 — `:hover` 스타일에 의존하면 안 됨 | 현재 RN/web 코드는 hover 거의 안 씀. 검증 필요 |
| **손가락 굵기** | 버튼이 폰 사이즈로 작으면 옆 버튼 같이 눌림 | PC 폰트 배율 확장이 main 에 있음(`e56bc99`). 검증하면서 보정 |
| **드래그 vs 스와이프** | 핀치줌은 OK, 일부 RN 제스처가 PC 터치 드라이버에서 다르게 인식 가능 | 실측 후 발견되는 것만 수정 |
| **인터넷 끊김** | Wi-Fi 끊기면 동기화 멈춤 | Firestore Web SDK 의 IndexedDB 캐시 활성화 → 잠깐은 버팀 |

## 🧠 자기 점검 — 다음에 떠올릴 것

1. **메모리 vs 실제 코드** — 메모리(`project_cloud_sync.md`) 는 "설계 결정", 실제 코드는 "이미 구현된 부분". 답하기 전에 git log + 핵심 파일 읽기로 현재 상태 확인 먼저. 이 세션 초반에 이걸 빠뜨려서 잘못된 7주 로드맵을 제시했고 사용자가 정정 지적함.
2. **사용자 환경 단순화 = 작업량 단순화** — 카드/프린터/금전함 빼니 6~8주 → 3~4일. 가정만으로 부풀리지 말고 환경부터 확정.
3. **키오스크 모드는 데스크톱 비유로 설명** — 사용자가 데스크톱 개발 출신이라 "AutoCAD 작업대" 같은 비유가 잘 통함.
4. **부팅 자동화 명령어 같은 건 학습 노트로 따로** — 세션 안에서만 보고 잊기 쉬움. 챕터 고정 + 학습 노트 두 단계 보존.

## 🔜 다음 작업

**1차 1번** — `utils/firebase.web.js` 를 Firebase JS SDK (web) 로 교체.

현재 RN 측이 사용하는 namespace API:
- `db.collection('x').doc('y').get()`
- `ref.onSnapshot(cb, errCb)`
- `auth().signInAnonymously()`

JS SDK v9+ modular API:
- `getDoc(doc(db, 'x', 'y'))`
- `onSnapshot(ref, cb, errCb)`
- `signInAnonymously(auth)`

**전략**: `firebase.web.js` 안에서만 modular API 호출하되, **밖으로 내보내는 함수 시그니처는 RN 버전과 동일** (`getFirestore()` 가 namespace 처럼 `.collection().doc()` 체인 가능한 객체 반환). 호출자(StoreContext, MenuContext, useOrderFirestoreSync) 코드 수정 없게.

작업 시작 전에 호출 패턴 한 번 훑고 어댑터 설계 그림부터 잡을 것.
