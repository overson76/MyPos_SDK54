## 🔄 일상 흐름

### 외부 (노트북)

```powershell
# 작업 시작
cd C:\MyProjects\MyPos_SDK54  # 또는 Ts_MyTool
git pull origin main          # 메인컴이 마지막에 push 한 것 받음

# === 별관 작업 권장 (한 주제 = 한 별관) ===
git worktree add ../MyPos_별관_새주제 -b feature/외부작업-새주제
cd ../MyPos_별관_새주제
# 작업...

# 작업 끝나면
git add . && git commit -m "feat: 외부 작업 ..."
git push origin feature/외부작업-새주제

# 또는 별관 안 쓰고 main 직접
cd C:\MyProjects\MyPos_SDK54
# 작업 + 커밋
git push origin main
```

**노트북에서 push = GitHub 만 갱신** (NAS 접근 X). 정상.

### 집 들어와서 메인컴

```powershell
cd C:\MyProjects\MyPos_SDK54
git pull origin main          # 노트북 변경 받음 (별관 브랜치라면 fetch + merge 또는 cherry-pick)

# 별관 작업 흡수 시 (메인컴에서)
git worktree add ../MyPos_별관_외부-새주제 feature/외부작업-새주제
# 검증 후 main 에 머지
cd C:\MyProjects\MyPos_SDK54
git merge feature/외부작업-새주제
git push origin main          # → GitHub + NAS 자동 양쪽 sync 🎉
```

→ **메인컴이 push 하는 순간 NAS 도 자동 갱신**. 따로 NAS 손댈 필요 없음.

## ⚠ 주의사항

### 1. 별관 (worktree) 작업 = 한 주제만
- 노트북에서 외부 작업 = 별도 브랜치 / worktree
- main 직접 수정도 가능하지만 충돌 위험 ↑
- 사장님 메모리 패턴: 별관 시작 → 작업 → main 머지 → 별관 정리

### 2. 메인컴이 NAS 동기화 책임자
- 노트북은 GitHub 만 (외부에서 NAS 접근 X)
- NAS 가 최신 되려면 메인컴이 한 번 push 해야
- 집 들어오면 메인컴에서 `git pull && git push` 한 번 권장

### 3. 같은 파일을 양쪽에서 동시 편집 X
- 노트북에서 작업 중 메인컴 작업 안 시키기 (또는 반대)
- 충돌 시 git merge 로 해결 가능하지만 복잡

### 4. `.env` 와 빌드 산출물 (`bin/`, `node_modules/`)
- gitignored — 클론 시 안 따라옴
- `.env` 는 보안 메신저 / USB
- 빌드 산출물은 노트북에서 새로 빌드

## 🛠 troubleshoot

### 노트북에서 push 안 됨 — "Authentication failed"
```powershell
gh auth status
gh auth refresh                # 또는 gh auth login 다시
```

### 메인컴에서 NAS push 안 됨 — "safe.directory" 경고
```powershell
git config --global --add safe.directory Z:/git/MyPos_SDK54.git
git config --global --add safe.directory Z:/git/Ts_MyTool.git
```

### 노트북에서 NAS 도 push 하고 싶다면 (집 들어왔을 때)
```powershell
# Z 드라이브 NAS 마운트되어 있어야
cd C:\MyProjects\MyPos_SDK54
git remote set-url --add --push origin Z:/git/MyPos_SDK54.git
# 이후엔 노트북에서도 양쪽 push (메인컴과 같은 상태)
```
