## 개념

### 결제 직전 receipt 캡처 — markPaid 후 데이터 변형 회피

```js
const orderSnap = autoPrint ? getOrder(id) : null;
const receiptData = autoPrint ? { items: orderSnap.items, total, paymentMethod, ... } : null;
markPaid(id, picked); // 또는 clearTable
if (receiptData) printReceipt(receiptData).catch(() => {});
```

`clearTable` 가 호출되면 `orders[id]` 가 사라지므로 호출 전에 데이터 캡처 필수. 호출 후 캡처하면 빈 receipt 출력.

### 요일별 집계 — `Date.getDay()` 0~6

JS 표준: 0=일요일, 1=월요일, ... 6=토요일. 한국 사람 친숙한 월~일 순서로 reorder 가능하지만 Date 인덱스 그대로 두고 label 로 표시 (`['일', '월', ..., '토']`).

### 영업일 카운트 — Set 중복 제거

```js
const dateSet = new Set();
dateSet.add(`${y}-${m}-${d}`);
```

같은 날 여러 건의 매출 → 영업일 1일. `dateSet.size` 가 영업일 수. 일평균 = monthTotal / totalDays.

### 운영 매뉴얼 = 진단 도구의 외부 인터페이스

이미 추가된 진단 도구(storeDiag) / 자동 업데이트(electron-updater) 카드들이 매뉴얼의 "트러블슈팅" 섹션과 1:1 매핑. 화면에 진단 정보 보여주고 → 매뉴얼이 어떻게 해석하는지 안내. 사장님이 우리(개발자) 없이도 자체 해결 가능.
