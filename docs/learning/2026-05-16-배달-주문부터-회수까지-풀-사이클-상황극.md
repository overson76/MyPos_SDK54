## 핵심 데이터 흐름 그림

```
[손님 전화] CID
   ↓
addressBook 매칭 → IncomingCallBanner (15s)
   ↓ [주문받기]
PENDING (cart + customerPhone + deliveryAddress)
   ↓ 메뉴 + [주문]
배달N (submitPendingAsDelivery → confirmOrder)
   ↓
🔊 speakOrder + 🖨️ 자동 영수증 + 📱 Firestore sync
   ↓ (조리)
[조리완료] readyAt 박힘
   ↓
✨ 회수 차수 진행 중 (oldest=ready, newest=paid 모두)
   ↓ (배달지 도착)
[결제] markPaid → history append
   ↓ (라이더 매장 복귀)
[🖨️ 출력] finalizeRound (Firestore 영구 + 마감)
   ↓ (라이더 회수)
[✓] markEntryReturned → returnedAt + 진행률
   ↓ (다음 결제/조리완료 배달)
2차 차수 자동 생성 (진행 중)
```

## 데스크톱 개발 비유

| MyPos 개념 | AutoCAD/Tekla 비유 |
|---|---|
| CID Webhook | AutoCAD `NETLOAD` 시점에 어셈블리 자동 로드되는 이벤트 hook |
| Firestore sync | Tekla 의 다중 사용자 모델 — 한 기기 저장 → 다른 기기 자동 반영 |
| 회수 차수 | AutoCAD 의 도면 시트 — 출력 = 마감 시점 snapshot 박힘 |
| 별칭 우선 라벨 | Tekla 의 사용자 정의 attribute 가 GUID 보다 우선 표시되는 패턴 |
| OrderProvider Context | AutoCAD 의 `Application.DocumentManager.MdiActiveDocument` 같은 단일 진실 |
| readyAt 박힘 | Tekla 의 `IsModified = true` flag |
| markPaid → history append | AutoCAD 의 "확정 후 새 도면 시트로 이동" |
