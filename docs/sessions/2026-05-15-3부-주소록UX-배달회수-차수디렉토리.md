## 핵심 기술 결정

1. **차수 = 출력 시점 snapshot** — entry.returnRoundId 박는 대신 차수가 ranked/unknown 결과 그대로 보관. 마감 차수는 정렬 토글 시 snapshot 의 distanceM 으로 resort, 출력은 snapshot 그대로.
2. **진행중 차수 = 가상** — Firestore X. 마지막 차수 createdAt 이후 entry 를 실시간 계산. 차수 번호 = (그날 마감 차수 수)+1.
3. **카카오 실거리 lazy fetch** — DeliveryReturnScreen 진입 시 백그라운드. 다음 렌더에 자동 실거리. fallback 시 "*" 마크.
4. **DeliveryMapModal mode 분기** — 기존 individual(배달 색깔별) 유지, sequential(매장→경유지→매장 한 줄) 신규. 회수에 적합.
5. **인덱스 바 가로** — 폰 가로 화면(932×430) + 매장 POS 일관성. iPhone 연락처 패턴(세로) 보다 매장 환경 우선.
6. **삭제 확인 dialog** — Firestore hard delete = 복구 불가. confirm 필수. 인덱스 드래그 중 실수 클릭 사고 방지.
