## 학습 포인트

### 1. OTA runtimeVersion 호환 — 매번 잊기 쉬운 함정

CLAUDE.md OTA 정책:
> `"runtimeVersion": { "policy": "appVersion" }` — 같은 `version` 빌드끼리만 OTA 호환.
> JS 만 변경 시 app.json version 안 올림. native 변경 시만 bump + 새 EAS 빌드.

1부 1.0.27 에서 "화면 하단 표시 정상화" 한답시고 app.json 1.0.2 → 1.0.27 sync 했다가 폰 OTA 끊김. 운영 룰:
- `package.json` version — electron-builder .exe build 식별자 (매번 bump OK)
- `app.json` version — RN runtime version (OTA 호환의 기준, **native 변경 없으면 절대 bump X**)
- 화면 하단 표시는 app.json 사용 → "MyPos v1.0.2 (12)" 가 정상. (12) 가 build number 라 매번 변함.

### 2. 폰 PanResponder 의 안전화 패턴

RN native 의 `onLayout(e)` 에서 `e.target` 은 number(reactTag) — measureInWindow 메서드 X.
RN web 의 `e.target` 은 DOM element — measureInWindow 도 메서드 X.
→ 둘 다에서 `e.target.measureInWindow` 안 작동.

해결:
```jsx
const viewRef = useRef(null);
<View
  ref={viewRef}
  onLayout={() => {
    if (!viewRef.current) return;
    try {
      viewRef.current.measureInWindow?.((x, y, w, h) => {
        layoutRef.current = { pageX: x, pageY: y, width: w, height: h };
      });
    } catch (err) {
      reportError(err, { ctx: 'layout.measure' });
    }
  }}
>
```

ref 자체에 measureInWindow 가 박힘 (RN native). RN web 은 optional chaining 으로 안전 skip.

- 결제 후 = paymentStatus 'paid' → "결제수단: 카드 / 결제상태: 결제완료"

장점:

- 향후 수정 시 한 곳만

### 4. Sentry breadcrumb 으로 silent 실패 진단

자동 출력 6단계 breadcrumb (autoprint.received / skip.no-printer / skip.toggle-off / policy / building / printResult). 사장님이 "자동 출력 안 됨" 보고 시 Sentry 의 마지막 breadcrumb 보면 어느 단계에서 멈췄는지 즉시 진단. 

PanResponder 핸들러도 동일 — try/catch 안에 reportError 로 stack 캡처. silent 실패가 silent 진단 가능.
