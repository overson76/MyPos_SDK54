## 핵심 학습

### LG U+ 070 SIP 서버 = ASN 인증

같은 LG U+ 라도 **광망(LG POWERCOMM AS17858) ↔ LTE 070 망(LGTELECOM AS17853)** 의 ASN 이 다르면 SIP 서버가 광망 출발지 패킷을 거부 (404). 이는 LG U+ 의 070 인터넷전화 보안 정책. 매장에서 광 인터넷 + LTE 070 동시 사용하면 두 회선의 출발지 IP 가 다른 ASN 에 속함.

### NSIS 인스톨러 자살 패턴

NSIS 의 `installer.nsh` customInit hook 에서 PowerShell 로 mypos*.exe 강제 종료 시 와일드카드(`MyPos*`)가 인스톨러 자신(`MyPos-Setup-X.X.X-x64`)도 매칭 → 시작 직후 자기 자신을 죽임 → 더블클릭해도 "아무 일도 안 일어남" 증상. 해결: `Where-Object ProcessName -notlike *Setup*` 단축구문 (PowerShell 3+).

### portable 도 electron-updater 동작

electron-builder 의 portable 빌드는 자체 electron-updater 가 작동해서 GitHub 의 NSIS Setup 을 매번 다시 다운로드 → "준비됨" 메시지 무한반복 + portable 종료 시 NSIS Setup 자동 실행 트리거로 portable 자체도 자동 종료되는 자살 패턴. 정상 패턴: portable 환경에서 `setupAutoUpdater` 호출 자체를 skip (`process.env.PORTABLE_EXECUTABLE_FILE` 가드).

### autoInstallOnAppQuit 의 위험

`autoInstallOnAppQuit:true` 는 영업 종료 후 .exe 닫을 때 자동으로 NSIS Setup 실행. 매장 운영 환경에서 NSIS uninstall 단계 hang 사고가 자동 적용 시점마다 반복돼서 1.0.11 부터 false 로 변경. 사장님이 명시적 시점에만 적용 (UI 의 "지금 적용" 버튼 추가는 1.0.14 예정).

### NSIS 인스톨러 디스크 요구사항

NSIS 인스톨러 정상 동작에 **C 드라이브 free 5 GB 이상** 필요:
- 7z 압축 해제 임시: ~1.7 GB
- 신규 파일 복사: ~1.7 GB
- 가상메모리(페이지 파일) 여유: 1~2 GB

매장 카운터 PC 처럼 가벼운 사양 + 옛 캐시 누적 시 디스크 부족 → NSIS 가 디스크 정리하면서 진행하느라 매우 느려짐 (5~10분 → 20분+).
