## 핵심 기술 결정

### 1. 합치기 패턴 (옵션 A) — default + Firestore
- 옛 동작: Firestore = 진실. 사장님 매장 처럼 Firestore 가 거의 비어있으면 default 사라짐
- 새 동작: default = baseline (코드 내장, 영구), Firestore = customize/신규 override
- 트레이드오프: default 의도적 삭제 불가 (옵션 C 의 deletedDefaultIds marker 가 필요해지면 추가)

### 2. write = 전체 item set (partial merge 금지)
- Firestore filter `(m)=>m.id!=null` 통과 보장
- `mergeItemForWrite` 가 default + current + partial + id 합쳐서 항상 전체

### 3. RN `<Modal>` 피하기 — absolute overlay
- iOS new arch + 중첩 TouchableOpacity 의 알려진 함정 (메모리 `project_modal_native_crash.md`)
- `<View style={position:absolute, top/left/right/bottom:0, zIndex/elevation:9999}>` 패턴
- AdminScreen PIN 모달과 동일

### 4. confirm 함수 Platform 분기
- `typeof window !== 'undefined'` 는 RN 에서 `true` (속음). `Platform.OS === 'web'` 으로 분기 필수
- 폰은 `Alert.alert` 로 native confirm

### 5. 격자 UX — 자리 강제 박기
- 빈 슬롯 = 박음 / 채워진 슬롯 = 교체 (옛 메뉴는 즐겨찾기에서 빠짐)
- confirm 없이 즉시 (☆ 다시 누르면 복원 가능)
- `placeIdInFavoriteGrid` 순수 헬퍼로 단위 테스트
