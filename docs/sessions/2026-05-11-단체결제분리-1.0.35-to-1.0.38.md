# 2026-05-11 — 단체 묶음 + 1인/테이블별 결제 분리 (1.0.35→1.0.38)

> 사장님 directive: "테이블을 합처놓고 주문을 누르면 어느 테이블의 주문인지 확인받게. 반대로 테이블별로 주문 받고 테이블을 합처도 각 테이블별 주문 내역과 금액은 남도록. 단! 1인결재인지 테이블별 결재인지도 상황이 달라지니. 단체와 관련한 명령이 있을때는 이부분도 고려해서 프로그램에 넣어줘."

매장 실무 두 시나리오(A: 단체 묶고 주문 / B: 테이블별 주문 후 합치기)와 두 결제 방식(합산 / 1인·테이블별 분리)을 한 번에 처리할 토대 + UI + API + 영수증을 4 단계로 나눠 완성.

## 단계별 진행

### Phase 1.0.35 — sourceTableId 토대 (commit `355c6a1`)

코드:
- `utils/orderReducer.js` · `addItem` 액션에 `sourceTableId` 인자 + `findIndex` 매칭 조건. `migratePendingCart` 시 `PENDING_TABLE_ID` 를 `toTableId` 로 자동 치환 (가상 ID 가 슬롯에 남지 않게).
- `utils/orderHelpers.js` · `normalizeSlots` 매칭 조건에 `(r.sourceTableId || '') === src` 추가. 옛 데이터끼리는 둘 다 빈 문자열로 매칭되어 그대로 합쳐짐.
- `utils/useGroups.js` · `createGroup` 안에 `stamp` 함수 — items/cartItems/confirmedItems 각 슬롯에 자기 `tableId` 박은 후 `mergeOrderParts` 호출.

테스트:
- `__tests__/orderReducer.test.js` — `addItem` sourceTableId 4케이스 + `migratePendingCart` 치환 2케이스
- `__tests__/orderHelpers.test.js` — `normalizeSlots` sourceTable 분리/합산/한쪽만 + `mergeOrderParts` sourceTable 유지 + 1.0.34 `largeQty/sizeUpcharge` 회귀 가드
- `__tests__/escposBuilder.test.js` — 1.0.33 간결화 회귀 정리 (매장명/부가세/카드/감사합니다 제거된 새 형식)
- `__tests__/payment.test.js` — markHistoryReverted 도입으로 추가된 "되돌림" CSV 컬럼 반영

### Phase 1.0.36 — 단체 묶음 후 메뉴 추가 UI (commit `8a0a61d`)

코드:
- `components/TableSourcePicker.js` (신규) — absolute overlay 패턴. 단체 멤버 큰 버튼 + 마지막 선택 ★ 강조.
- `screens/OrderScreen.js` — `useOrders` 에서 `getGroupFor` 추가. 메뉴 카드 `onPress` 흐름을 `handleMenuPress(item)` 로 추출. group leader 면 모달 띄움, 일반이면 즉시 `applyMenuAdd`. `lastSourceByGroup` state 로 그룹별 마지막 선택 기억.
- `utils/OrderContext.js` — `addItem` 시그니처 확장 (4번째 인자 `sourceTableId`).

### Phase 1.0.37 — 결제 분리 (commit `a01f7ba`)

코드:
- `utils/orderReducer.js` — 새 액션 `'orders/clearTableBySource'`. 해당 sourceTable 슬롯만 items/cartItems/confirmedItems 에서 제거. 모두 비면 테이블 자체 제거.
- `utils/orderHelpers.js` — `groupItemsBySource` / `computeSubtotalsBySource` helper. 분리 결제 모달의 손님별 금액 표시 + 회계 사무소 CSV 분리 컬럼용.
- `utils/OrderContext.js` — `clearTableBySource` wrapper. history entry 에 `sourceTableId` + `isPartial: true` 박음. 단체에 속하면 leader 로 통일 + 모든 슬롯 비면 group dissolve. 컨텍스트 value 에 `clearTableBySource` / `computeSubtotalsBySource` / `groupItemsBySource` 노출 + fallback 도 동일.
- `components/GroupPaymentSplitPicker.js` (신규) — 손님별 소계 ScrollView + 합계 + "전체 합산 결제" (파란) / "테이블별 분리 결제" (주황) 두 큰 버튼. absolute overlay.
- `screens/TableScreen.js` — "결제하기" 진입점에 `startPayment` 추출 — 단체이면 split picker 띄움. `handleSplitStart` 가 `splitPayQueue` 큐로 첫 손님 `PaymentMethodPicker` 띄움 → 결제 후 `clearTableBySource` + 다음 손님 자동 진행. 마지막 결제 시 자동 종료. 모달 취소 시 큐도 해제.
- `screens/OrderScreen.js` — "후불" 진입점도 동일 분기. "선불" 은 그대로.

테스트:
- `__tests__/orderReducer.test.js` — `clearTableBySource` 4케이스 (해당 손님만 제거 / 모두 비면 테이블 제거 / 없는 테이블 noop / 옛 슬롯 fallback)
- `__tests__/orderHelpers.test.js` — `groupItemsBySource` / `computeSubtotalsBySource` 5케이스

### Phase 1.0.38 — 분리 결제 영수증 (commit `7f327fd`)

코드:
- `utils/escposBuilder.js` — `buildReceiptText` 가 `receipt.isSplit` + `sourceTableLabel`/`sourceTableId` 받아서 헤더 "📋 테이블" 줄 아래 "👤 분리 결제 손님: <라벨>" 한 줄 추가. 합산 결제는 영향 X.
- `screens/TableScreen.js` + `screens/OrderScreen.js` — `receiptData` 에 `sourceTableLabel` 도 같이 박음. OrderScreen 은 `resolveAnyTable` import.

테스트:

## 배포

- **라이브 URL** (Cloudflare Workers) — `npm run deploy:web` (PC 카운터 즉시 갱신)
- **폰 OTA** — `eas update --branch production` (다음 앱 시작 시 자동 다운로드)
- **.exe 자동 업데이트** — `electron-builder --publish always` (GitHub Releases, 다음 시작 시 적용)

`runtimeVersion: { policy: "appVersion" }` 정책상 `app.json` 의 version 은 1.0.2 그대로 유지 — 폰 OTA 호환 보장. `package.json` 만 1.0.38 로 bump.
