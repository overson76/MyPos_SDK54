## 핵심 기술 결정

1. **글로벌 lastCallPhone (5분 TTL)** — useIncomingCall.web.js module-level state. CID 알림 받을 때마다 갱신. OrderScreen 의 모달 트리거 조건/currentPhone fallback. *어떤 슬롯에 진입해도* 최근 CID 데이터 따라옴. 5분 후 만료로 옛 phone 자동 사라짐.

2. **시뮬 자동 entry 생성** — App.js MainApp onSimulateCall wrapper. simulateCall + upsertEntryFromOrder 같이. 시뮬 단골이 *진짜 entry* 가 되어 어디서든 매칭. 사장님이 ✏️ 또는 ⛔ 로 시뮬 잔재 정리.

3. **needsAliasPrompt 다단계** — alias OR entry alias OR phone OR lastCallPhone. *식별 정보가 어디든 있으면 모달 skip*. confirmOrder 자동 sync 가 entry 만듦. 사장님 의도 "그냥 주문만 들어가면 되잖아".

4. **mergeSameAliasPhoneOnlyEntries** — isPhoneOnlyLike 헬퍼로 `__phone:` key 또는 `(주소 미입력)` label 둘 다 식별. 사장님 매장의 옛 형식 잔재 (수동 생성 / 옛 코드) 모두 통합. migrationRanRef 제거로 매 entries 변경 시 호출 (idempotent).

5. **5km 매장 반경 절대 강제** — geocodeAddress 가 center 받으면 keyword 검색을 searchKeywordNearby 로 분기 + radius 5km. 전국 검색 절대 금지. 잠복 311km entry 영구 차단. 부팅 청소 effect 가 옛 잠복 entry 좌표/거리 reset.

6. **Toast Context + Web Fixed Position** — RN-Web 의 fixed position 으로 페이지 스크롤 무관 viewport 상단 고정. 5초 자동 사라짐. 사장님이 어떤 화면 어느 스크롤이든 즉시 보임.
