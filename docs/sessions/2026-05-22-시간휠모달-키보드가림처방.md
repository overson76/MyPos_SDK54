- ⏳ **폰 실 환경 미검증** — preview 검증은 코드 import + jest + 콘솔 error 0 까지. 가로 키보드 회피와 휠 스와이프 실제 느낌은 폰에서 확인 필요. 사용자 명시: "회귀하더라도 모두 저장 반영".

## 핵심 기술 결정

1. **순수 JS 휠 선택 (네이티브 모듈 X)**
   - `react-native-date-picker` / `@react-native-community/datetimepicker` 모두 네이티브 코드 포함 → 빌드 14 강제, OTA 못 보냄.
   - FlatList + `snapToInterval` + `decelerationRate="fast"` 로 휠 느낌 구현.
   - 마이포스의 OTA 우선 운영 가치와 부합.

2. **저장 형식 100% 호환**
   - 휠 confirm 결과를 기존 `"420"` (3자리) / `"1130"` (4자리) string + boolean isPM 로 변환.
   - `parseDeliveryTime` / `formatShort12h` 등 유틸 무수정 — 영수증·예약 알림·일일 보고서 코드 그대로.

3. **5분 간격 스냅**
   - 배달 시간은 보통 5분 단위로 말함 ("4시 20분에 보내주세요" / "오후 4:25 까지").
   - 1분 간격은 휠이 60칸 → 사장님 스와이프 횟수 ↑ + 5분 단위로 어차피 정리됨.

4. **모달 패턴 — Modal 미사용**
   - 이미 마이그레이션된 `absolute overlay` 패턴 유지 (`project_modal_native_crash`).
   - KAV 를 overlay 안 backdrop 다음에 끼우는 식 — sheet 가 키보드 영역만큼 자연스럽게 위로 밀려 올라감.

5. **PinManageModal 처방 제외**
   - PinEntry 가 자체 숫자 버튼 패드 (TextInput 미사용) → OS 키보드 안 뜸 → 처방 불필요.
