# 세션 Q&A — 2026-05-07 (되돌리기 탭 분리)

주제: 직전 세션(94430cd) 에서 도입한 되돌리기 안전망을 운영 중 빠르게 찾도록 상부 탭 메뉴에 별도 "되돌리기" 탭을 추가. 한 화면 안에서 결제완료 / 조리완료 두 모드를 토글하고 최신순으로 정렬해서 보여줌.

핵심 결론: **새 탭 = 운영 도구**. 기존 RevenueScreen / KitchenScreen 의 되돌리기 버튼은 그대로 유지(역호환). 새 탭은 그 두 곳을 한 데 모아 사장님이 영업 중에 즉시 접근 가능. 도메인 로직 변경 없음 — `revertHistoryEntry` / `undoMarkReady` 그대로 호출.

## Q&A

**Q1. 결제완료/조리완료 되돌리기가 별도의 탭에 있으면 좋겠어. 주문현황 오른쪽에 (되돌리기) 탭을 하나 추가해서 내부에서 결제완료/조리완료 나눠주고 클릭하면 완료된 최신순부터.**
→ 3 파일 변경:
1. [App.js:64](App.js:64) — `TAB_KEYS` 에 `'되돌리기'` 끼워넣음 (주문현황 ↔ 관리자 사이)
2. [App.js:341-348](App.js:341) — `<UndoScreen />` 패널 mount (`activeTab !== '되돌리기' && paneHidden` 패턴)
3. [screens/UndoScreen.js](screens/UndoScreen.js) 신규 — 두 모드 세그먼트 토글 + 최신순 리스트 + 행마다 빨간 `↶ 되돌리기` 버튼

**결제완료 모드**: `revenue.history` 에서 `reverted` 제외, `clearedAt` 내림차순. 행마다 테이블/시각/선후불·결제수단/항목/주소/합계 + `↶ 되돌리기` → `revertHistoryEntry(entry.id)`.

**조리완료 모드**: `orders[]` 중 `status==='ready'` 인 살아있는 테이블, `readyAt` 내림차순. 행마다 테이블 라벨/완료 시각·경과분/항목 + `↶ 되돌리기` → `undoMarkReady(tableId)`.

각 탭 라벨 옆에 건수 (`결제완료 되돌리기 (N)` / `조리완료 되돌리기 (M)`) 즉시 표시 — 사장님이 누적량 한눈에.

확인 다이얼로그 / 실패 사유 매핑 / `reverted` 플래그 등 도메인 로직은 RevenueScreen 의 기존 핸들러와 동일. 코드 중복은 있지만 **새 화면은 운영 도구라 의존성을 최소화** — 두 화면이 서로 의존하지 않음.

**Q2. 바로 적용할거야**
→ 즉시 작성 + 코드 검증. 브라우저 자동 검증은 worktree 에 node_modules 없어서 (Expo 의존성 ~수분 설치 필요) 스킵. 코드 구조 / import 경로 / 도메인 함수 시그니처 확인까지만.

**Q3. 세션 종료 하고 빌드 까지 최종 진행**
→ 표준 사이클:
1. 세션 노트 + README 갱신 (이 파일)
2. feat 커밋 → main 머지 → 1.0.14 → 1.0.15 버전 bump → release 커밋 → push
3. 빌드/배포 (eas update + deploy:web + electron:build --publish) — 가능한 항목만 시도, 실패 시 정확한 원인 보고

JS 전용 변경 (네이티브/의존성 X) → OTA 가능. 매장 PC .exe 는 라이브 URL 우선 로드 패턴이라 deploy:web 한 번이면 다음 시작 시 새 번들 자동 로드 (.exe 자체는 재빌드 불필요).
