## 근본 원인 — 별칭 소실 2중 경로

1. **데이터 손실** (`utils/orderReducer.js` `orders/addItem`): 주문 객체를 *필드 나열식*으로 재구성하면서 `deliveryAlias`/`deliveryPhone` 을 빠뜨림 → CID 로 박은 별칭/전번이 **메뉴 담는 순간** 증발.
2. **표시 격차** (`screens/TableScreen.js`): 주소록 lookup 이 `if (isDelivery && deliveryAddr)` 게이트 → 배달은 주소→전번으로 별칭 재획득(손실 가려짐), **포장/예약은 lookup 자체가 없어** `order.deliveryAlias` 가 비면 그대로 텅.
   - `moveOrder` (자리이동) 자체는 `{ ...state[fromId] }` 얕은 복사라 모든 필드 보존 — 범인 아님. 손실은 이미 addItem 에서 발생.

## 알려진 문제 / 미해결 이슈

- **`addItem` 이 배달 알림 플래그(`deliveryAlerted10`/`deliveryAlerted5`)를 리셋** — 메뉴 추가 시 둘을 보존 안 하고 죽은 `deliveryAlerted`(단수)만 씀. 메뉴 담을 때마다 10/5분 전 알림이 재발화할 수 있는 잠복 버그. **별도 태스크로 분리**(이번 변경엔 미포함). 의도된 동작인지 `useDeliveryAlerts` 확인 후 수정.
- 포장/예약 lookup 은 *전번*이 있어야 작동 — 순수 워크인(전화 없는) 포장은 여전히 별칭 없음(정상, 식별 정보 자체가 없음).
