## Q&A (질문 순서)

**Q1. 배민 연동 가능한가? 금전적 리스크?**
→ API 비공개 (역방향 엔지니어링 위험). 공식 배민오더 단말기 방식이 현실적. 단기 도입은 보류 권장.

**Q2. 카드 단말기 결제 구현 가능한가?**
→ KIS-NAGT + OCX(ActiveX COM) 방식. C# 브릿지 콘솔앱 → Electron child_process → OCX 호출로 구현. simulate 모드 default, 환경변수 한 줄로 실 결제 전환.

**Q3. KIS POSPORTAL 우회 경로 (공휴일, 전화 불가)?**
→ POSPORTAL 회원가입(POS업체, 사업자 요건 없음) → 자료실 접근. 에이전트 설치 파일 + 매뉴얼(kisdevice.gitbook.io) 무료 확보.

**Q4. KIS 자료실 에이전트 파일 + 매뉴얼 내용?**
→ OCX 3가지 모드(일반/Event/Unit). inTranCode="NV", inTradeType="D1/D2", inAgentIP="127.0.0.1", inAgentPort=1515. KisAgent_Setup_3460.zip + dotNetFx40 + VS2008 재배포 다운로드.

**Q5. C# 브릿지 + IPC 모듈 구현 완료?**
→ 완료. electron/payment/bridge/ (Program.cs, Models.cs, .csproj) + electron/payment/kis.js (simulate/bridge) + utils/kisPayment.web.js + PaymentMethodPicker 통합.

**Q6. 자동 업데이트 배너 — 메인 화면에 진행률 표시?**
→ components/UpdateBanner.js 신규. downloading 중에만 상단 배너 등장, downloaded 시 "준비 완료" 안내, dismiss 버튼. App.js 최상단 통합.

**Q7. GitHub 리포 public 전환 — 왜 필요?**
→ private 리포에서 매장 PC가 GitHub Releases.atom 접근 못 해 404 → 자동 업데이트 동작 안 함. PowerShell API 한 줄로 public 전환.

**Q8. sha512 mismatch 에러 원인?**
→ portable + NSIS 두 타겟이 같은 파일명으로 publish → GitHub에서 덮어씀 → latest.yml 해시 불일치. builder.config.js에서 artifactName 분리로 해결 (v1.0.4).

**Q9. UpdateBanner dismiss 버그 (알겠습니다 ✕ 안 먹힘)?**
→ version이 빈 문자열이면 dismiss 조건 통과 못 하는 코드 버그. sentinel 값(__no_version__) + currentVersion 비교 추가 (v1.0.6, 후 v1.0.7).

**Q10. 1.0.5 sendSync freeze — MyPos 몇 초 후 꺼짐?**
→ preload.js의 ipcRenderer.sendSync가 sandbox: true 환경에서 freeze → 즉시 crash. async invoke 형으로 변경 (v1.0.6 긴급 fix).

**Q11. "Failed to uninstall old application files" 반복 에러?**
→ NSIS oneClick: false에서 기존 파일 잠금 해제 실패. Program Files\MyPos 이중 설치 + electron-updater autoInstallOnAppQuit 루프. oneClick: true로 변경 (v1.0.8) + 수동 폴더 삭제로 정리.

**Q12. CID — SIP 연결 시도?**
→ MYPOS_SIP_HOST=sip.lgdacom.net, USER=tmpid7133, PASS=7133 설정. SIP REGISTER 실패 (무반응). INVITE crash fix는 cid.js 콜백 전체 try/catch 강화 (v1.0.7). 내일 LG U+ 1544-0001 전화로 실제 SIP 계정 정보 확보 필요.

**Q13. 주문 내역 글씨 작아서 보기 힘듦?**
→ cartItemName fp(8)→fp(12), cartItemPrice fp(7)→fp(11), qtyNum fp(7)→fp(12) 등 전체 1.5배. OrderScreen.styles.js (별관 cherry-pick).

**Q14. 결제하기 버튼 뜰 때 최종 가격 안 보임?**
→ badgesWrap의 결제하기 버튼이 tileTopLeft(금액) 밀어내는 구조. tilePayBtn 안에 총금액 직접 표시 (69,000원 + 💰 결제하기 세로). TableScreen.js + styles (별관 cherry-pick).

## 학습 포인트

### KIS OCX 연동 구조
KIS 결제 API = 32비트 OCX(ActiveX COM). Node.js/Electron 직접 호출 불가 → C# 브릿지 패턴.
stdin JSON → C# 콘솔앱 → OCX 호출 → stdout JSON → Electron.

### NSIS 설치 에러 패턴
`oneClick: false` = 기존 파일 잠금 해제 실패 → "Failed to uninstall" 반복.
`oneClick: true` = silent install → 에러 없음. 자동 업데이트 앱엔 true 권장.

### SIP REGISTER vs INVITE
REGISTER 실패: 전화 와도 앱 변화 없음.
REGISTER 성공 + INVITE 처리 crash: 전화 오면 앱 꺼짐.
→ 전화 오면 꺼짐 = SIP 연결은 됐다는 증거, 코드 문제.
→ 전화 와도 무반응 = SIP 서버 또는 계정 정보 문제.
