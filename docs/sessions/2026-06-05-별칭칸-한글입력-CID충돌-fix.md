# 2026-06-05 — 별칭칸 한글 입력 + CID 전화 작업 중 충돌 fix

## 한 줄 요약
어제 별칭 모달 fix 후속 두 건: ① PC 별칭칸에 한글이 안 써지던 IME 버그, ② 전화 수신이 작업 중 주문을 가로채 뒤죽박죽·중복·배달탭 꼬임. 둘 다 처방·배포 완료, 내일 매장 검증 대기.

## 이슈 1 — PC 별칭칸 한글 안 써짐

| 항목 | 내용 |
|---|---|
| 증상 | PC 카운터 별칭칸에 영어·숫자는 되는데 **한글만** 안 써짐 |
| 원인 | 4부에서 넣은 `returnKeyType`/`blurOnSubmit`/`onSubmitEditing` 이 RN-Web(Electron)에서 한글 IME 조합 키(keyCode 229)를 가로채 조합을 끊음. (영어/숫자는 조합이 없어 멀쩡) |
| 격리 근거 | 다른 한글칸(주소록 별칭·메뉴명)은 submit 핸들러가 없어 정상 → submit 핸들러가 범인 |
| 처방 | 한글이 정상인 AddressBookModal **검색칸과 동일 형태로 단순화** (submit 핸들러 3개 + maxLength 제거, `onChangeText={setAlias}`). `components/AliasPromptModal.js` |
| 트레이드오프 | 폰에서 별칭→전화칸 엔터 넘김 편의 제거 (한글 입력 우선) |
| 배포 | 커밋 `507f9b9`. deploy:web `1fafe449` + OTA `0fe9f3fc` |

## 이슈 2 — CID 전화가 작업 중 주문 가로챔 (심각)

| 항목 | 내용 |
|---|---|
| 증상 | 메뉴/주소/연락처 입력 중(완료 전) 전화 오면 주문 **뒤죽박죽 + 중복 + 배달탭 꼬임** |
| 원인 ① | `submitPendingAsType` 1단계 `migratePendingCart` 가 작업 중 PENDING 카트를 빈 배달칸으로 끌고 가며 전화 손님 정보 덮어씀 (A 담는 중 B 전화 10초 → A메뉴+B정보, A 카트 사라져 재입력=중복) |
| 원인 ② | `findEmptySlotForType` 이 발신자 정보만 있는 "🕐 주문대기" 슬롯을 메뉴 없다고 "빈 칸"으로 재판정 → 다음 전화가 같은 칸 덮어씀 = 배달탭 꼬임 |
| 처방 ① | `submitPendingAsType` 에 `migrateCart` 스위치. 전화 흐름(10초 자동 + "주문받기")=**false** → 작업 중 카트 안 건드림. OrderScreen "주문" 버튼은 기본 true 유지 |
| 처방 ② | `findEmptySlotForType` 점유 판정에 발신자(전화/별칭/주소)·예약(시간/인원) 포함 |
| 배포 | 커밋 `3313176`. jest +5 (592). deploy:web `0e4ec01e` + OTA `f9b05a74` |

## 핵심 기술 결정 (왜)
- **한글 IME**: controlled TextInput 에 submit 핸들러/maxLength 가 RN-Web 한글 조합을 깰 수 있음 → 정상인 검색칸과 동일 형태가 안전.
- **CID stash ↔ 작업 카트 분리**: 전화 자동처리는 **절대 PENDING 카트를 건드리지 않는다**(migrateCart false). 발신자 정보만 빈 슬롯에.
- **슬롯 점유 판정 = 메뉴 + 발신자 + 예약 정보**: "주문대기"/"빈 예약" 슬롯이 다음 전화·예약에 덮어써지지 않게.
