## 핵심 기술 결정

### keyboardVerticalOffset 값 기준
- **모달 = 60**: 모달 헤더(타이틀+닫기 ~44) + 보더/패딩(~16). input 을 키보드 위로 60 더 띄움.
- **화면 = 50**: SafeArea top inset (노치 ~44) + 여유.

### 좁은 변경 원칙 (root 직접 교체 X)
SettingScreen / AdminScreen 의 root `<View>` 를 KAV 로 바꾸지 않고 *메인 ScrollView 만* 감쌈. 이유: root 에 박힌 absolute overlay 모달들 (favoriteModal / PIN 모달 등) 이 부모 KAV 의 padding 영향 받아 어긋나는 부작용 방지.

### 기존 자산 재사용
- 공용 컴포넌트 [components/KeyboardAwareContent.js](../../components/KeyboardAwareContent.js) — 이미 표준 패턴 박힘 (이번엔 신규 적용 안 하고 기존 KAV 에 offset 만 보강).
- AddressBookPanel 의 row onLayout y offset 자동 scrollTo (2026-05-23 작업) — 그대로 유지 + paddingBottom 보강.

## 알려진 문제 / 미해결 이슈

- `keyboardVerticalOffset` 값 60/50 은 *경험값*. 실제 폰 검증 후 미세 조정 가능 (너무 크면 컨텐츠 위로 잘림, 너무 작으면 가림 잔존).

## 사고 A: 주소록 검색 결과 가림 (offset 처방으로 안 잡힌 케이스)

**원인**: offset 처방은 *입력칸* 은 키보드 위로 올렸지만, 검색칸(상단 toolbar) 아래의 *검색 결과 리스트* 는 여전히 키보드 아래 깔림. landscape 화면(430px)이 좁아 키보드(~300px) 뜨면 리스트 가시 영역이 거의 0.

**처방 1차 (6f28ef2)** — `Keyboard.addListener('keyboardDidShow')` 로 키보드 높이 직접 구독 → 리스트 ScrollView `contentContainerStyle` 에 그만큼 `paddingBottom` (스크롤로 결과 확인). landscape 의 KeyboardAvoidingView 부정확성 우회.

**처방 2차 (4df8dcf)** — 1차만으론 부족 (리스트 영역 거의 0이라 스크롤 여지 적음). 사장님 제안대로 **키보드 닫기 명시적 액션** 추가가 핵심:
- 검색 input `returnKeyType="search"` + `onSubmitEditing={() => Keyboard.dismiss()}` — 키보드 검색키(엔터)로 닫기.
- 키보드 떠있을 때(`kbHeight > 0`) 검색칸 우측 파란 **"✓ 완료"** 버튼 → 누르면 `Keyboard.dismiss()`.
- 키보드 닫으면 리스트가 화면 전체로 복구 → 결과 다 보임. 1차 동적 paddingBottom 은 보조로 유지.

파일: [components/AddressBookPanel.js](../../components/AddressBookPanel.js) — `kbHeight` state + Keyboard listener effect + searchDoneBtn 스타일.

## 사고 B: "새 버전 있음" 영구 표시 (표기 버그)

**원인**: [utils/buildInfo.js](../../utils/buildInfo.js) 의 `APP_VERSION` 이 `'1.0.52'` 하드코딩인데, `package.json`(1.0.53) + GitHub Releases 최신(v1.0.53, 5/24 생성)과 어긋남. 5/24 에 1.0.53 릴리스 publish 시 이 상수 갱신을 빠뜨림. `compareVersions('1.0.52','1.0.53') = -1` → 항상 "새 버전 있음". **실제 새 빌드 필요한 게 아니라 표기 버그.**

**처방 (6f28ef2)**: `APP_VERSION = '1.0.52'` → `import { version as PKG_VERSION } from '../package.json'; export const APP_VERSION = PKG_VERSION;`. 배포마다 자동 동기화 → 다신 안 어긋남. (Metro/Expo JSON import 지원, jest 통과 확인.)
