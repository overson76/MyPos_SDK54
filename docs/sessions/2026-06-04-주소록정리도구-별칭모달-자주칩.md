## 핵심 기술 결정

### 정리 도구 survivor 우선순위
`entryScore` = 진짜 주소(+100000) > 별칭(+10000) > 주문횟수(×10) > firstSeenAt tie-break. 정보 풍부한 entry 를 살리고 나머지 흡수.

### 같은 번호 = 안전(ON) / 비슷 상호 = 판단(OFF)
같은 전화번호는 같은 손님 확실 → 기본 통합. 비슷한 상호("탑마트" vs "신평 탑마트")는 다른 가게일 수 있어 사장님이 "같은 곳" 체크 시만.

### 폰에서 정리 → PC 자동 반영
정리는 setAddressBook → Firestore `stores/<sid>/addresses` write → PC onSnapshot 자동 sync. PC 라이브 URL 코드 갱신 없이도 데이터 정리 반영.

### 컴포넌트 검증 — import 스모크
react-test-renderer 미설치(CLAUDE.md 명시). jest-expo 가 모달 import 시 JSX 트랜스폼+import 체인 평가 → 문법/import 오류 잡음. 실제 렌더는 폰/PC.
