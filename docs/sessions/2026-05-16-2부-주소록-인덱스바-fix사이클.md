| 6 | iPhone native | Pressable 부모가 child PanResponder touch 가로챔 | View + responder system 필요 |

## 핵심 교훈

1. **환경별 RN API 차이를 명시적 분기로 처리** — Platform.OS === 'web' / native 분기는 좌표 / DOM API / scroll 모두 적용
2. **PanResponder + Pressable 중첩 시 capture 옵션 명시** — RN native 의 touch 경쟁에서 우선권 확보
3. **scrollIntoView 는 RN-Web 전용** — native 통합 시 ScrollView.scrollTo + onLayout offset 패턴
4. **viewport 계산은 backdrop padding 차감 필수** — `maxHeight: '90%'` 가 padding 외 90% 라는 사실 검증
5. **iOS Safari touch-action 흡수 회피** — `touchAction: 'none'` 명시 안 하면 가로 스와이프가 페이지 동작에 흡수
6. **한 fix 가 다른 환경 깰 수 있음** — pageX 우선 적용이 native 시작점 어긋남 만든 사고 (8차에서 분기 fix)
