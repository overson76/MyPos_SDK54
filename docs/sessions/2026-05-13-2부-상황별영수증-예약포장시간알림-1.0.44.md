## 핵심 기술 결정

### 데이터 모델 — 기존 `deliveryTime` 필드 재사용
- 후보: `scheduledTime`/`scheduledTimeIsPM` 으로 리네임 vs 기존 그대로
- 선택: **기존 그대로** — 마이그레이션 0, 코드 변경 최소. 의미가 약간 어색해도 모든 호출부가 이미 사용 중

### 알림 hook — 이름 유지, 내부만 일반화
- `useDeliveryAlerts` 이름 그대로 두고 type 별 음성 분기. 파일명 변경 안 함 — 호출부 0 변경

### ESC/POS 명령 — string 안에 inline
- buildReceiptText 가 string 반환 유지. 명령 바이트(`\x1B\x21\x30` 등)는 ASCII 범위라 iconv-lite EUC-KR 변환 시 그대로 보존
- `print.js` 의 `buildTextBytes` 가 wrap → 프린터가 명령 그대로 처리

### 손님번호 = 연락처 통합 처리
- 사용자 요청에 "손님번호" + "연락처" 둘 다 명시됐으나 데이터는 주소록 `phone` 하나뿐 → 한 줄로 통합 (`손님 010-1234-5678`)
- 분리 출력 필요 시 별도 데이터 필드 추가 후 영수증 빌더에 라인 한 줄 추가하면 가능

## 정식 배포 (이 세션에서 적용)

매장 영업에 즉시 반영하기 위한 흐름:

1. `git push origin claude/...` (별관 push)
2. `git checkout main && git merge claude/...` (fast-forward)
3. `git push origin main`
4. `npm run deploy:web` — Cloudflare 자동 빌드 함정 회피, 로컬 .env Firebase 키 inline 보장
5. `eas update --branch production` — 폰 OTA 발행

PC 카운터 `.exe` 자동 업데이트(Phase 3)는 native 변경 없으므로 생략. `.exe` 가 다음 시작 시 라이브 URL 에서 새 번들 받음.
