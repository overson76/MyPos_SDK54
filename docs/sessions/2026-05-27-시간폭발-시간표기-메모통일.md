## 핵심 기술 결정

### 1. 시간 폭발 이중 처방 (호출부 fix + 방어 가드)

호출부만 고치는 건 *증상 처방*. 방어 가드도 추가해서 *root cause 클래스* 영구 차단:

- `formatShort12h(parsed)` — 객체 외 입력 들어와도 `''` 반환
- `formatKorean12h(parsed)` — 동일

향후 다른 호출자가 같은 실수해도 화면 안 멎음. jest 8 케이스로 회귀 차단.

### 2. KitchenScreen 시간 표기 — 회색 톤 단일 + 아이콘 분기

색 분기 (배달 빨강 / 예약 파랑 / 포장 초록) 대신 회색 톤 단일 + 아이콘으로만 구분. 카드 안에 이미 deliveryBar (빨강) / requestBar (주황) 존재 — 시간 바까지 색 분기하면 시각 노이즈. 통일성 우선.

### 3. KitchenScreen 메모 — `itemNameWrap` 내부 들여쓰기

별도 행 → 메뉴 박스 내부로. 옵션 라인 (`itemOptLine` 파랑) 과 같은 위계지만 메모는 amber 톤 + 좌측 borderLeft 로 구분. 어느 메뉴의 메모인지 시각적으로 명확.

미사용 `itemMemoLine` / `itemMemoText` / `itemMemoTextPhone` 즉시 제거 — backwards-compat 잔재 X (사용자 메모리 `Don't add backwards-compatibility hacks`).

### 4. TableScreen 배달 시간 표기 — 분기 확장 + deliveryPrimary 분리

기존 분기 `(reservation || takeout)` 안에 deliveryPrimary 표시 + 시간 표시 묶음.
→ 배달은 위쪽 라인에 deliveryAddr 별도 표시 있음 — 시간만 분기 밖으로 독립.
→ deliveryPrimary 표시 (별칭/전번 prefix) 는 reservation/takeout 분기 안에 유지.
→ 시간 표시는 모든 type 에 대해 분기 밖에서 처리 — 일관성.
