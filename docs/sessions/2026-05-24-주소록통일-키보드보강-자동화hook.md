## 핵심 기술 결정

### 1. **AddressBookModal 의 phones array UI 이식** — AddressBookPanel 패턴 그대로

처음부터 두 곳을 *통일된 헬퍼/컴포넌트* 로 분리하는 안도 검토했으나, 두 파일의 UI 컨텍스트가 다름 (Panel = 풀스크린 + 거리 컬럼 / Modal = absolute overlay + 인덱스 바). 공통 추출 시 prop drilling 복잡 → *코드 중복 허용 + 같은 UX 패턴* 선택. 향후 둘 다 한 더 큰 변경이 들어오면 그 때 분리 재검토.

### 2. **자동화 hook 3중 안전망**

| 단계 | 작동 시점 | 효과 |
|---|---|---|
| SessionStart | 세션 시작 | 다른 PC 작업 자동 가져오기 |
| PreToolUse | git push 직전 | push 시점 fetch 재검증 → 사이에 다른 PC push 한 것 차단 |
| Stop | 응답 마무리 | 미커밋 / 미푸시 / 별관 미머지 경고 |

세 hook 이 *서로 다른 시점* 에 작동해 *어느 한 단계가 우회되어도* 다른 단계가 cover. 어제 force-update 사고 같은 경우 — 사장님이 다른 PC 에서 hook 없이 push 한 경우 *Windows PC 시작 시 SessionStart hook 이 잡음*.

### 3. **NAS multi-url 제거** — secrets 역할로만

NAS Z: 가 *secrets/EAS state 동기화* 의도였는데 git remote 까지 multi-url 로 같이 박혀 *부수효과로 git mirror* 역할. 결과 *GitHub vs NAS 진실 불일치* 사고 (어제). 정리:
- `origin` = GitHub 단일 push (단일 진실 소스)
- `nas` = Z:/git/MyPos_SDK54.git (수동 push 만 — `git push nas main` 명시 시)

다른 도메인 작업이라 분리 commit. 회귀 또는 rollback 시 독립 처리 가능.
