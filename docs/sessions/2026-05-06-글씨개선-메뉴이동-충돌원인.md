# 세션 Q&A — 2026-05-06 (2차)

주제: 글씨 개선 OTA 재배포 + 폰 메뉴 이동 기능 시도 + 충돌 원인 파악

---

## Q&A (질문 순서)

**Q1. 결제하기/글씨 개선이 아이폰에 적용 안 됨. 메뉴 이동도 안 됨.**
→ deploy:web 만 하고 eas update를 빠뜨림. 이후부터 세션 정리 시 두 가지 모두 필수. OTA 즉시 재배포.
→ 메뉴 이동은 PC(웹) 전용으로만 구현됐었음 — 폰에는 별도 구현 필요.

**Q2. 테스트 시 테이블 데이터가 매출에 잡히지 않나?**
→ 오늘 preview 클릭 테스트(화면 이동/모달 열기)는 Firebase write 없음 → 매출 영향 없음.
→ 앞으로 규칙: 빈 테이블로만 테스트, 테스트 전/후 상태 보고.

**Q3. 폰 메뉴 이동 기능 추가 시도 — 1차 충돌**
→ onLongPress + 스타일 추가 시도. 앱 크래시 발생.
→ 롤백 (revert + OTA).

**Q4. 충돌 원인 1차 추정: styles.js fp() scope 문제?**
→ makeStyles 함수 내부에 정확히 추가 후 재시도 (2차).

**Q5. 2차도 동일 충돌 발생. 진짜 원인은?**
→ Sentry 확인: FIRInvalidArgumentException — Nested arrays are not supported.
→ setCategorySlot() 호출 → writeMenuRowsFs() → Firestore에 2D 중첩 배열 저장 시도 → iOS 네이티브 SDK 크래시.
→ 웹(PC)은 Firestore JS SDK가 처리 → 에러 없음. iOS만 크래시.
→ 6일 전부터 있던 기존 버그. 메뉴 이동 기능이 setCategorySlot 호출해서 트리거.

**Q6. 세션 정리 약속 — 세션 종료 시 반드시:**
→ `deploy:web` (라이브 URL) + `eas update --branch production` (폰 OTA) 둘 다 해야 함.

## 핵심 학습

### Firestore iOS 네이티브 SDK — 중첩 배열 불가
Firestore에 `[[a, b], [c, d]]` 형태 저장 시도 → iOS native SDK: FIRInvalidArgumentException.
Firestore JS SDK는 처리 가능하지만 RN Firebase native는 불가.
해결: 저장 전 flat 변환 필요.

### 세션 종료 배포 체크리스트
```
deploy:web    → 카운터 PC(웹 라이브 URL) 반영
eas update    → 폰(iOS + Android OTA) 반영
```
둘 다 해야 모든 기기 동기화.
