## 핵심 기술 결정

### 1. **isFocused 모드 — 전용 화면 전환**
편집/추가 시 사장님이 *오직 폼만* 봐야 한다는 사장님 의도. inline edit (list 안 한 row 가 편집 폼) → 별도 큰 영역 폼 + list/searchBar 숨김. 폰 가로 키보드 환경 자연스러움.

### 2. **doSubmitOrder 함수화 + override order 인자**
AliasPromptModal 콜백 후 *비동기 state 갱신 차단* 위해 override 인자 — 음성/영수증에 *입력한 alias 즉시 반영*.

### 3. **카카오 로컬 키워드 검색 — 일반전화 fallback**
카카오 로컬 API 의 `/v2/local/search/keyword.json` 가 *전화번호 문자열* 도 매칭. 가게는 카카오에 *전화번호 등록* 하므로 일반전화/070 검색 가능. 휴대폰 (010) 은 개인 번호라 미등록 — 명시적 skip 으로 *낭비 X*.

### 4. **CID phone-only entry 자동 통합**
upsertEntryFromOrder / mergePhoneIntoEntry 가 `__phone:digits` entry 발견 시 자동 삭제 — *사장님이 별도 정리 작업 안 해도 됨*. 신규 전화 → CID 자동 등록 → 사장님 주문 → 자동 entry 통합 → 단일 source.

### 5. **순차 배포 syntax error 발견 + 재배포**
도중 deploy:web 의 expo export 가 syntax error 잡음. jest 통과 (import 안 함) 라 syntax 위험 차단 안 됐음. 향후 *pre-commit hook* 또는 *prettier* 같은 lint 도입 검토.
