# 2026-05-08 · 영수증 프린터 (SEWOO SLK-TS400 USB) 통합 — 매장 맥북 크로스빌드 + Phase 3 적용 미해결

> 매장 카운터 PC 의 SEWOO SLK-TS400 영수증 프린터를 USB 로 잡고 MyPos 와 통합. 매장 맥북에서 macOS → Windows .exe 크로스빌드 + GitHub Releases v1.0.16 publish 까지 성공. 단 매장 카운터 PC 의 1.0.16 적용은 자동 업데이트 (install on quit) 와 NSIS 수동 installer (MyPos cannot be closed) 모두 막힘 — 다음 세션에서 해결.

## 핵심 Q&A

**Q1. 세션 마무리 명령("세션 정리/세션 종료/모두 저장") 이 매장 맥북이든 집 윈도우 PC든 동일하게 동작하게 기억해놔.**
**A1.** 메모리(이 PC 한정) + `docs/collaboration-rules.md` 6번 갱신 (양쪽 PC 자동 전파). 트리거 동의어 5종 인정 + "푸시까지 반드시 완료" 명시. 커밋 `ff60843`. 이번 세션 본 종료 명령으로 자체 검증.

**Q2. OPOS 셋업이 뭐고 깔아야 하나?**
**A2.** OPOS = OLE for Retail POS (Microsoft 의 COM 기반 POS 표준 어댑터). MyPos 는 `node-thermal-printer` 가 Windows 프린트 스풀러에 raw ESC/POS 바이트 보내는 방식 → **OPOS 불필요**. SEWOO Windows Driver (Elite 4.70, 180dpi, 12MB) 만 깔면 충분.

**Q3. 매장 카운터 PC 에서 git pull + npm install + 빌드 해도 되나?**
**A3.** 안 됨. 매장 카운터 PC = 운영 전용 (.exe 더블클릭만). dev 도구 깔면 운영 환경 오염 + 영업 중 빌드 부하 위험. 빌드는 dev 머신(집 윈도우 PC 또는 매장 맥북)에서 → .exe 만 매장 PC 로 옮기는 분리.

**Q4. GitHub 푸시하면 윈도우 ↔ 맥 dev 머신끼리 코드 자동 공유라, 어디서 작업해도 같지 않나? 매장 맥북에서 빌드 + 배포까지 가능?**
**A4.** **맞다.** git 은 소스 코드 + package-lock.json 동기화. 빌드 산출물(.exe) 은 git 미공유라 별도 절차(USB or GitHub Releases 자동 업데이트) 필요. 매장 맥북도 정상 dev 머신 — `electron-builder` 가 macOS 호스트에서도 wine 자동 다운로드해서 Windows .exe 크로스빌드 가능. 이번 세션에서 직접 검증 ✅.

**Q5. 매장 맥북에서 .env 가 없는데 어떻게 빌드?**
**A5.** 본관(`/Users/KimMinJung/MyPos_SDK54/.env`) 에 .env 있어 worktree 에 심볼릭 링크 생성. `.env` 자체는 gitignored 라 안전. 본관 갱신 시 worktree 도 자동 반영.

**Q6. 자동 업데이트 다운로드 완료(`downloaded` 상태) 됐는데 .exe 재시작해도 1.0.16 적용 안 됨. 카운터 PC 가 느려서?**
**A6.** **속도 무관.** 진짜 원인 = electron-updater 의 `autoInstallOnAppQuit: true` 가 **정상 종료**(Electron 의 `app.quit()` 흐름) 일 때만 동작. 사장님이 X 버튼/창 닫기로 닫아도 일부 케이스에서 메인 프로세스가 백그라운드에 남거나 트리거 안 됨. CLAUDE.md Phase 3 의 "강제 quitAndInstall X" 안전 정책 영향. 다음 세션 진단 항목.

**Q7. NSIS installer 직접 실행해도 "MyPos cannot be closed" 반복.**
**A7.** 같은 라인 — MyPos.exe 프로세스가 살아있어 NSIS 가 잠금 해제 못 함. `installer.nsh` 의 사전 hook (1.0.11 시점 fix) 이 1.0.16 빌드(macOS 호스트 NSIS 컴파일)에서 정확히 박혔는지 미확인. 정공법 = 매장 PC 재부팅 후 다른 앱 일체 켜기 전 installer 실행. 그래도 실패하면 `MyPos-1.0.16-x64-portable.exe` 우회.

**Q8. SEWOO SLK-TS400 의 적합 드라이버는?**
**A8.** **Windows Driver (180dpi)** = `SEWOO TECH Elite Printer Driver 4.70` (12 MB). 페이지에 같이 보이는 "Windows Driver" 38.3 MB 는 Thermal 시리즈 통합본 (203dpi 포함, 더 무겁고 범용). SLK-TS400 사양표가 180 DPI 명시. 12MB Elite 가 정합. 설치 후 Windows 프린터 표시 이름 = `SLK-TS400`.

## 학습 포인트 — 매장 맥북 macOS → Windows .exe 크로스빌드

| 항목 | 결과 |
|---|---|
| 호스트 OS | macOS (Apple Silicon) |
| 타겟 OS | Windows x64 |
| `electron-builder` 버전 | 26.8.1 |

| nsis 자동 다운로드 | ✅ (nsis-3.0.4.1, 1.3MB) |
| `node-thermal-printer` native 컴파일 | 문제 없음 (거의 순수 JS) |
| 산출물 동작 검증 | ⚠️ NSIS installer 의 사전 hook 동작은 미해결 (다음 세션 항목) |

**필수 조건**:
- `.env` 가 worktree 루트에 있어야 함 (Firebase / Sentry inline). 본관 → worktree 심볼릭 링크 패턴.
- `EXPO_PUBLIC_FIREBASE_API_KEY` 가 dist/ 번들에 inline 됐는지 grep 검증 (CLAUDE.md 함정 2).
- `gh auth token` 으로 GH_TOKEN 임시 추출 → publish 시점 1회 한정 사용. 영구 환경변수 오염 X.
- `electron-builder --publish always` 가 OOM/timeout 으로 exit 137 가능 → 우회 = 빌드만 하고 `gh release create` 로 직접 업로드 (background 권장).
