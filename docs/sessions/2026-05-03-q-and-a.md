## Q&A (질문 순서)

**Q1. 오늘 있었던 이슈를 상황별로 설명할게. 원인을 찾아봐**
→ TestFlight 1.0.2 (새 native 빌드) + .exe 1.0.2 autoUpdater 첫 부팅 = Firebase 익명 UID 새로 발급 → Firestore `permission-denied` 폭발. 장소(매장) 자체는 원인 아님. "매장에서 처음 켠 시점"이 새 빌드 첫 실행 타이밍이었을 뿐.

**Q2. 1~4번 니가 직접 확인할 수 있는 부분 상세히 확인해봐**
→ `electron-dist/latest.yml`: 5/2 17:35에 `.exe 1.0.2 publish` 흔적 확정. 라이브 URL 번들 해시 ≠ 로컬 dist 해시 (어젯밤 `deploy:web` 안 했음 확인). GitHub API는 private 리포라 인증 없이 불가.

**Q3. Sentry 화면 한 장 캡처 — 어쩌라는건지 자세히 설명**
→ 사장님 폰 자체 화면 X. 사용자가 `sentry.io` 웹 콘솔에 로그인해서 오늘 이슈 보는 것. Chrome 자동화로 내가 직접 연람.

**Q4. Sentry URL 직접 열어서 확인해봐**
→ JAVASCRIPT-7: `firestore/permission-denied` 102 events / 4 users / iOS 100%. Tags — `dist: 12 (buildNumber 12 = 새 EAS 빌드)`, `runtime_version: 1.0.2`, `is_embedded_launch: true (새 빌드 첫 실행)`. 원인 확정.

**Q5. 지금 할 수 있는 세팅 다 해놓고 내일 매장에서 문제없이 바로 사용 가능한 걸 원해**
→ 코드 7개 변경 + commit `c523683` + EAS OTA + deploy:web 완료. 자가 복구 흐름 (lastStore 캐시 + 노란 카드), 폰 지도 활성화, Sentry release 동적화.

**Q6. (사진) MyPos v1.0.2 (12) iPhone/iPad**
→ 너 폰도 1.0.2 = OTA 호환. 이번 변경 폰으로 즉시 배포 가능 확인. 라이브 URL 검증 완료.

**Q7. A로 가고 문제 발생 가능성 있으면 허락 요청해**
→ commit → EAS OTA → deploy:web 순서로 진행. 자가 복구 노란 카드 Chrome 자동화로 검증. 배포 완료.

**Q8. 지도 떴어. 그리고 정리해줘**
→ OTA 지도 작동 확인. 학습 노트 `docs/learning/2026-05-03-uid-loss-diagnosis-and-self-recovery.md` + 메모리 3개 (신규 2 + 갱신 1) + git push 완료.

**Q9. 분명히 할 게 있어. 내가 사장이자 개발자야**
→ 메모리 `user_role.md` 추가. 사장 = 개발자 = 본인. 이후 "사장님께 안내" 같은 분기 제거.

**Q10. 노란 카드와 PIN 부분 명확히 설명해줘. 내 아이폰에서 뭘 해놔야 하는거?**
→ 노란 카드는 미리 만들 수 없음 — 사고 나야 자동 등장. **지금 해놔야 할 것은 매장 PIN 설정 하나뿐.** 관리자 → 시스템 → 매장 관리 → 수익 PIN 설정.

**Q11. 가족 사업인데 직원/대표 구분 필요한가? 다 대표로 해도 무방해?**
→ 가족 사업이면 모두 owner 가 정답. 단 현재 처음 가입 경로는 staff 만 가능 → A 옵션 (대표로 가입 옵션 추가) 결정.

**Q12. A로 가자 (대표 가입 옵션)**
→ `JoinConfirmView` 에 [직원으로 가입] / [대표로 가입(PIN)] 두 옵션 토글 추가. commit `37cb00d` + EAS OTA `9b96948d` + deploy:web 완료. 이제 매장 코드 + PIN 만 알면 누구든 owner 로 즉시 입장.

**Q13. 카운터 PC CID 작동 원리 — 같은 와이파이 조건 안에서 되는거지?**
→ 유선/Wi-Fi 무관. **같은 LAN 이면 OK.** 매장 가입 복구 = `useCidHandler(storeId)` 자동 호출 = SIP REGISTER = CID 즉시 시작.

**Q14. 어제까지 사용했다고 누가 그래? ㅋㅋ**
→ 인정. CID 코드는 5/1에 들어갔지만 매장 가입 자체가 끊겨 있어서 실전 작동 한 번도 안 됨. 내일이 진짜 첫 시도.

**Q15. 두 PC 동시 켜놓고 전화 오면 어느 게 받아?**
→ LG U+ 정책상 last-write-wins (마지막 등록 PC) 가능성 높음. 하지만 어느 PC 가 SIP 받든 Firestore broadcast 로 **모든 기기에 팝업 + TTS** → 운영 영향 X.

**Q16. 카운터 PC 한 대만 연동 확인 + 전화해보고, 괜찮으면 그때 주방 PC 켜보면 되는 거 아냐?**
→ 정답. 코드 변경 없이 단계별 검증. 카운터 → SIP 작동 → 주방 → SIP 동시 등록 관찰.

**Q17. 내가 한 질문만 따로 보거나 과정별로 정리된 것만 보는 건 어떻게?**
→ A) transcript 검색, B) Q&A 정리 파일, C) 단계 흐름 파일, D) Stop hook 자동화. B+C 조합 추천.

**Q18. 만들자. 모든 세션에서 똑같이 작동하게 해줘**
