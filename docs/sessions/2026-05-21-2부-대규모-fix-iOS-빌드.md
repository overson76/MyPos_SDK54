## 핵심 기술 결정 — 왜 이렇게 했나

### 1. `KAKAO_KEY` lazy getter (Firebase 패턴 통일)

옛 패턴 `const KAKAO_KEY = process.env.X || ''` 가 *module 최상위 const + `||` fallback* 조합에서 babel-preset-expo inline transformer 가 치환 실패. Firebase 처럼 *함수 호출 시 직접 참조* 패턴이 inline 안전. *진단 결정적*: 빌드 산출물에 Firebase 키 (12자) 1번, Kakao 키 0번.

### 2. `useGroups` 의 `groupModes` 별도 state — groups 구조 보존

옛 groups: `{ [leaderId]: [tableId1, tableId2, ...] }` 배열. 객체로 확장하면 모든 사용처 변경 필요. 대신 *별도 state* `groupModes: { [leaderId]: 'shared' | 'split' }` 추가. 기존 코드 영향 0 + 옛 데이터 자동 'shared' (옛 동작).

### 3. split 모드는 createGroup 시 *merge 안 함*

shared: 옛 동작 그대로 — mergedLeader 계산 + reducer 가 멤버 slot 삭제 + leader 에 합산. *split*: mergedLeader 계산 X + reducer 도 호출 X (`if (mode === 'shared')` 가드). 각자 자기 state 그대로 유지 → 각 슬롯 진입 시 자기 메뉴만 표시.

### 4. CID 자동 dismiss 부활 (어제 정책 뒤집기)

어제 (1부) 는 "주문 확정 시 dismiss" 와 "15초 자동 dismiss" 가 충돌해서 *자동 dismiss 제거*. 오늘 사장님 새 룰 "10초 후 자동 빈 d 슬롯 stash" 도입하면서 *자동 dismiss 부활*. submitPendingAsType('delivery') + dismissIncomingCall() 짝으로 호출 — 충돌 없음.

### 5. iOS Runtime Version 1.0.1 vs 1.0.2 진단

`runtimeVersion.policy = 'appVersion'` — app.json version 과 정확 일치 필요. Android 는 5/3 빌드 1.0.2 라 호환. iOS 는 5/2 빌드 1.0.1 라 *영원히* 1.0.2 OTA 못 받음. 새 iOS 빌드 + TestFlight 업로드 + 사장님 설치 1회 = 영구 해결. EAS 빌드 13 → TestFlight 자동 submit.

### 6. Windows file lock race condition (wrangler ENOENT)

오늘 deploy:web 두 번 wrangler ENOENT 에러. dist 파일은 정상 존재. preview 서버 동시 동작 또는 다른 프로세스 lock. 해결: preview 끄고 *wrangler 단독 재실행* — 정상. 추후 deploy-web.sh 에 retry 로직 추가 검토.
