## 핵심 기술 결정

1. **AI 추천 — 외부 API 없이 로컬 통계로 시작** → 비용 0, 매출 데이터 활용도 ↑. Claude API 옵션은 운영 후 검토.
2. **배달 그리디 + 페어 거리 캐싱** → 5건 이하 묶음에 충분, 카카오 일일 5,000건 무료 한도 안에서 안정.
3. **DeliveryRouteCard.getDistanceFn prop 주입** → 시연 데모가 mock 거리 함수 주입해서 카카오 API 의존 빼고 항상 결과 표시 가능. 운영은 default getDrivingDistance.
4. **단골요청은 동적 lookup** — order 에 박지 않고 렌더 시점에 deliveryAddress → entry lookup. 주소록 변경 시 즉시 반영, 옛 주문도 최신 요청 노출.
5. **추천 카테고리는 categoryRows 격자 시스템에 통합** → 메뉴 그리드 렌더 코드 재활용, 별도 컴포넌트 불필요. 편집/드래그는 `isRecommendation` 가드로 차단.
6. **머지 후 PR 분리 가능** — 4개 커밋 (`c4e1acf`, `093262a`, `2f0e00d`, `5d363e7`) 각각 별도 머지 가능한 단위.
