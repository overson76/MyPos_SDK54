증상:
- 폰(아이폰)에서 "주문" 탭 누르면 `TypeError: undefined is not a function`

- Sentry 자동 보고됨
- 부수: 주소록 모달 검색이 자음만 보이고 조합 안 됨 (첫 크래시로 부분 렌더 깨진 후일 가능성)

- 사용자 확인 정보: 폰에서 발생 + 주문 탭 진입 시 크래시 + iOS 업데이트 있었음

- 미확인: 폰의 정확한 버전 표시

가설:
| 가설 | 가능성 | 처방 |
|---|---|---|

## 핵심 기술 결정

1. **DeliveryRouteCard candidates 를 순수 함수로 분리** — Jest 가 `@sentry/react-native` ESM transform 못 함 → 컴포넌트에서 직접 export 못 함 → `utils/routeCandidates.js` 별도 파일. 분리 후 회귀 방지 가능
2. **returnedAt 데이터 모델** — 옛 차수(필드 없음) 도 동작하도록 빈 값 = 미회수 인식. 자동 마이그레이션 X (데이터 무결성)
3. **회수 진행률 fillAllReturned 의 immutability** — 이미 회수된 entry 의 timestamp 유지(되돌리기 방지). 새 entry 만 now 박힘. Jest 검증
4. **phone 매칭 union 정책** — customerAddressKey + customerPhone 둘 다 들어오면 union. "신규 손님(첫 주문) 이 같은 phone 으로 다른 주소 시킬 때" 도 매칭 성립
5. **시연 모드 default OFF** — 일반 사장님 혼란 방지 + 시연 가치 유지 (한 클릭 ON)
6. **트랜잭션 회피** — useDeliveryRounds 의 returnedAt 변경도 기존 `finalizeRound` 와 같은 `.set(round)` 패턴. RNFB v24 + iOS 새 아키 트랜잭션 크래시 처방 유지
