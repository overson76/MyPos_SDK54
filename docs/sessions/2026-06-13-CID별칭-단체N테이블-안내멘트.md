## 핵심 기술 결정 (왜)

- **`setDeliveryContact` null = "변경 없음"** — 별칭만 갱신하는 호출이 CID stash 가 박은 전화번호를 지우지 않도록. 도장 함수가 "보강"이지 "덮어쓰기"가 아니라는 의미를 reducer 레벨에 못 박음(단위테스트 6종이 회귀 방지).
- **시뮬과 실 CID 의 매칭 단일화** — 시연 흐름이 실전과 다르면 검증이 거짓이 됨. `matchCidEntry` 한 함수로 두 경로가 같은 결과를 내게 강제.
- **단체 확정을 버튼으로 분리** — "2번째 클릭=즉시 확정"은 N개 확장과 양립 불가. 클릭은 누적(선택/해제)만, 확정은 명시적 [단체 확정(N)] 버튼으로. 기존 확정 경로 재사용이라 reducer/결제 변화 0.
