## Q&A (질문 순서)

**Q1. CID 사진들 분석해서 정보 확인**
→ 12장 분석. IP450S 정보 확보:
- 전화기 IP: `192.168.10.100`
- 게이트웨이/DNS: `192.168.10.254`
- 서브넷: `255.255.255.0`
- MAC: `A8:E5:39:91:1E:AC`
- 모델: `IP450S` by 모임스톤

**Q2. 192.168.219.1 공유기로 SIP 비번 찾을 수 있나?**
→ NO. LGU_2C60 공유기에는 SIP 정보 없음. 192.168.10.100 (전화기) 웹 관리자에 있음. 단 카운터 PC와 망이 분리돼서 직접 접근 불가.

**Q3. 비밀번호 시도 — `Sk2TTqtN2hv6Z54`**
→ 실패. setx는 저장만 할 뿐 검증 안 함. MyPos 재시작 후 전화로만 검증 가능.

**Q4. setx 후 `성공: 지정한 값을 저장했습니다` = 비번 검증?**
→ 아님. 저장만 성공이라는 뜻. 실제 비번 검증은 SIP REGISTER 시도할 때.

**Q5. 전화 안 해보고 SIP 연결 확인 가능한가? IP 다른 건 OK?**
→ `netstat -an | findstr 5060` 으로 SIP 리스너 동작 여부만 확인 가능. IP는 외부 SIP 서버에 등록되니 카운터 PC와 전화기 IP가 달라도 OK.

**Q6. 왜 전화선이 CNR에 꽂혀있고 LGU_2C60에 직접 연결 안 했나?**
→ CNR-L1000 = LTE 회선용 VoIP 게이트웨이. 광인터넷에 꽂아도 의미 없음. IP450S만 광망에 옮기는 건 가능 (단, LG U+ 가입 형태 따라 다름).

**Q7. CNR-L1000이 두 개인데 하나는 전화기, 하나는 카드결제?**
→ 정확. 매장 안정성 위한 분리 구성:
- CNR-L1000 #1 → IP450S (인터넷전화)
- CNR-L1000 #2 → KIS-PAD (카드결제)
- LGU_2C60 → 카운터 PC (광 인터넷)

**Q8. centrex.uplus.co.kr 관리자 페이지에서 비번 찾을 수 있나?**
→ YES! 핵심 발견:
- SIP 도메인이 `lgdacom.net` 이 아니라 **`premium_c_347878.lgdacom.net`** (이래서 SIP 등록 실패!)
- 비번 직접 변경 가능 (보여주지는 않음, 새 비번 설정만)
- 새 비번 설정: `dmscks501*`

**Q9. 새 비번 + 도메인 업데이트 후 재시작했는데 5060 포트 무반응?**
→ 환경변수 OK + 매장 데이터 정상 = storeId OK. 그럼 sip.start()가 실행 안 됐거나 실패.
→ 가장 유력: **sip 패키지가 빌드된 .exe 안에 누락됨**.
→ production .exe 라 main process 로그 안 보임. 1.0.9 진단 빌드 필요.

## 핵심 학습

### LG U+ 인터넷전화 SIP 도메인은 매장별 다름
센트릭스 가입 시 매장 코드가 도메인에 포함됨:
```
일반: lgdacom.net
센트릭스: premium_c_347878.lgdacom.net
```
잘못된 도메인 사용 시 SIP REGISTER 자체가 무응답.
센트릭스 관리자 페이지(centrex.uplus.co.kr)에서 정확한 도메인 확인.

### 매장 네트워크 분리 구성
LG U+ 매장은 보통 3개 회선 분리:
- LTE #1 (CNR-L1000) → 인터넷전화 (IP450S)
- LTE #2 (CNR-L1000) → 카드결제 (KIS-PAD)
- 광 인터넷 (LGU_2C60) → 일반 인터넷 (PC, Wi-Fi)

이 분리로 한 회선 끊겨도 다른 서비스 계속 동작.
하지만 카운터 PC가 LTE 망의 전화기/카드결제와 직접 통신하려면:
- 정적 라우팅 추가
- 또는 USB-LAN 어댑터로 직접 연결
- 또는 IP450S/KIS-PAD를 광망으로 임시 이동

### Production .exe 진단 한계
production Electron .exe 에선 main process 의 console.info/warn/error 가
사용자에게 안 보임. 정확한 진단을 위해선:
- IPC로 상태를 렌더러에 노출
- 관리자 화면에 진단 카드 추가
- 또는 Sentry 로 에러 자동 보고
