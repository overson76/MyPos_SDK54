## 2. 제품 개요

소형 매장(배달 + 홀 동시 운영)을 위한 멀티 플랫폼 POS 솔루션.

| 항목 | 내용 |
|---|---|
| 플랫폼 | iOS / Android / Web / Windows .exe (Electron) |
| 매장 동기화 | 폰 + 태블릿 + 매장 PC 실시간 Firestore onSnapshot |
| 결제 단말 | KIS-NAGT (한국정보통신) 통합 (코드 완성, 가맹 후 활성) |
| CID(발신자 표시) | LG U+ Centrex Rest API Webhook 직접 통합 |
| 영수증 | SEWOO USB 프린터 (PowerShell winspool 인라인 우회) |
| 자동 업데이트 | electron-updater + GitHub Releases |
| 오프라인 캐시 | Cloudflare 다운 시 .exe 안 dist/ 자동 폴백 |
| 회계 | 부가세 분리 + 한국어 CSV (Excel BOM 안전) |

## 7. 솔직한 평가

### 7.1 강점

- **본인 매장용 자체 POS 로는 이미 초과 충분.**
- 시장 평균 대비 차별 기능 **3개 이상** (CID 자동 배달, 카카오 시간 슬롯, LG U+ Webhook).
- 1인 개발 + 직접 매장 운영의 **이중 검증** — 실 운영자 입장의 운영 흐름을 코드에 직접 반영.
- 멀티 플랫폼 (폰/태블릿/매장 PC/웹) 단일 코드베이스 — 유지보수 효율 매우 높음.
- 데이터 클라우드 동기화 (Firestore) — 매장 이전/멤버 추가 등 운영 변경 즉시 반영.
- 자동 업데이트 + 오프라인 캐시 — 매장 운영 사고에 강한 구조.

### 7.2 약점 (사업화 시 고려)

- **POS 시장은 영업/지원/단말 인증/카드사 가맹 등 진입 장벽이 매우 큼.** 기술이 아니라 영업력 게임.
- 1인 개발이라 **고객 지원 인프라** 부족 (전화/원격 지원/24시간 대응).
- **카드 단말 인증** (한국 VAN 사 가맹) 별도 진행 필요. 본인 매장은 KIS-NAGT 진행 중.
- 마케팅 / 영업 비용 — 매장 1곳 모집에 30~50만 영업비 + 첫 설치/교육 시간.

### 7.3 가장 합리적 다음 단계

1. **사장님 매장 1년 무사고 운영** — 운영 검증 (현재 진행 중)
2. **친한 사장님 매장 1~2곳 무료 베타** — 케이스 스터디
3. **5~10곳 paid 검증** — SaaS 모델 안정화
4. **본격 사업화 결정** — 영업/지원 인프라 동반 구축

## 10. 권장 사용 시나리오

### 10.1 한 매장 자체 운영
- **매장 PC 1대** + 사장님 폰 + 직원 폰 (선택)
- 월 0원 (Firebase Free Tier 안에서 충분)
- 영업 시간: 10:00~20:00 (일요일 휴무 기준 가능)

### 10.2 소규모 가맹점 (5~10곳)
- 각 매장 PC + 사장님/직원 폰
- 사장님이 본사 입장에서 모든 매장 통계 / 매출 조회 가능 (Firestore admin)
- 월 운영비 매장 1곳당 3,000~10,000원 (Firebase 사용량 기반)

### 10.3 SaaS 사업화
- 멀티 매장 관리 콘솔 별도 개발 필요 (현재 단일 매장 가정)
- 결제 시스템 (Stripe / 토스 페이먼츠) 추가
- 고객 지원 인프라 (Zendesk / 자체) 구축
